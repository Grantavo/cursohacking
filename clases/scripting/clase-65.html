Scripting en Bash [2-6]

Duración: 28:39 · Proyecto 1: Buscador HTB

Añadimos funciones auxiliares esenciales: print_table para formatear resultados, print_error para mensajes de error, validación de entrada, conexión a la API de HTB con curl y procesamiento JSON con jq.

Función print_error

Mensajes de error estandarizados con color rojo:

print_error() {
  local msg="$1"
  echo "${RED}${BOLD}[✗] Error:${RESET} ${RED}$msg${RESET}" >&2
}

print_success() {
  local msg="$1"
  echo "${GREEN}${BOLD}[✓]${RESET} ${GREEN}$msg${RESET}"
}

print_info() {
  local msg="$1"
  echo "${BLUE}${BOLD}[i]${RESET} ${BLUE}$msg${RESET}"
}

Función print_table

Formatea resultados en una tabla con columnas alineadas usando printf:

print_table() {
  local sep="┌──────────────────────────────────────────────────────┐"
  echo "$sep"
  printf "│ %-20s │ %-15s │ %-10s │\n" "Nombre" "IP" "OS"
  echo "├──────────────────────────────────────────────────────┤"
}

print_table_row() {
  local name="$1" ip="$2" os="$3" difficulty="$4"
  local color="$RESET"
  case "$difficulty" in
    easy)   color="$GREEN" ;;
    medium) color="$YELLOW" ;;
    hard)   color="$RED" ;;
    insane) color="$MAGENTA" ;;
  esac
  printf "│ %-20s │ %-15s │ ${color}%-10s${RESET} │\n" "$name" "$ip" "$os"
}

Conexión a la API de HTB

Hacemos la petición con curl y procesamos la respuesta con jq:

fetch_machines() {
  local url="${API_URL}/list"
  local response
  local http_code

  print_info "Consultando API de HackTheBox..."

  response=$(curl -s -w "%{http_code}" \
    -H "Authorization: Bearer ${HTB_TOKEN}" \
    -H "User-Agent: HTBSearch/1.0" \
    "${url}")

  http_code="${response: -3}"
  response="${response%???}"

  if [[ "$http_code" -ne 200 ]]; then
    print_error "HTTP $http_code al consultar la API"
    return 1
  fi

  echo "$response"
}

Validación y procesamiento con jq

Extraemos los campos relevantes del JSON:

parse_machines() {
  local json="$1"
  echo "$json" | jq -c '.results[]' 2>/dev/null | while read -r machine; do
    local name os ip difficulty
    name=$(echo "$machine" | jq -r '.name')
    os=$(echo "$machine" | jq -r '.os')
    ip=$(echo "$machine" | jq -r '.ip // "N/A"')
    difficulty=$(echo "$machine" | jq -r '.difficulty // "unknown"')
    echo "${name}|${ip}|${os}|${difficulty}"
  done
}
💡 jq -c vs jq: La opción -c (compact) imprime cada objeto JSON en una sola línea, lo que facilita iterar con while read. Sin -c, jq formatea con saltos de línea y sangría.

Flujo completo

Combinamos fetch y parse en la función principal:

main() {
  print_banner

  local json
  json=$(fetch_machines) || exit 1

  print_info "Procesando resultados..."
  parse_machines "$json" > /tmp/htb_results.txt

  local count
  count=$(wc -l < /tmp/htb_results.txt)
  print_success "Se encontraron ${count} máquinas"
}
