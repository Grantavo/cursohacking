Lectura e interpretación de permisos [1-2]

Duración: 9:52 · Sección 2: Temario

Los permisos en Linux son el mecanismo fundamental de seguridad. Controlan quién puede leer, escribir o ejecutar cada archivo y directorio del sistema.

Los tres permisos básicos: rwx

SímboloSignificadoNúmero
rRead (lectura)4
wWrite (escritura)2
xExecute (ejecución)1
-Sin permiso0

Las tres categorías

Cada archivo tiene permisos para tres grupos:

  • Usuario (u) — el dueño del archivo
  • Grupo (g) — los miembros del grupo al que pertenece
  • Otros (o) — el resto de usuarios

Interpretar ls -l

La salida de ls -l muestra los permisos así:

$ ls -l /etc/passwd
-rw-r--r-- 1 root root 1234 May 10 12:00 /etc/passwd

Desglose:  -  r w -  r - -  r - -
           │  │ │ │  │ │ │  │ │ │
           │  │ │ │  │ │ │  │ │ └── otros: lectura
           │  │ │ │  │ │ │  └──── otros: sin escritura
           │  │ │ │  │ │ └─────── otros: sin ejecución
           │  │ │ │  │ └───────── grupo: lectura
           │  │ │ │  └─────────── grupo: sin escritura
           │  │ │ └────────────── grupo: sin ejecución
           │  │ └──────────────── dueño: escritura
           │  └────────────────── dueño: lectura
           └───────────────────── tipo: - (archivo normal)
🔍 Ejemplos de interpretación:
  • -rwxr-xr-- → Archivo normal, dueño puede rwx, grupo puede rx, otros solo r
  • -rw------- → Solo el dueño puede leer y escribir
  • -rwxr-xr-x → Todos pueden ejecutar, solo el dueño escribe
  • drwxr-xr-x → Directorio (la d indica que es un directorio)

Permisos en directorios

En directorios, los permisos tienen significados específicos:

  • r — listar el contenido (ls)
  • w — crear o eliminar archivos dentro
  • x — acceder al directorio (cd) y atravesarlo
⚠️ Sin permiso x en un directorio, no puedes acceder a él aunque tengas r. Necesitas ambos: r-x para entrar y listar.

Ejercicio práctico

# Crea un archivo y examina sus permisos
touch prueba.txt
ls -l prueba.txt

# Cámbialos y observa
chmod u+x prueba.txt
ls -l prueba.txt

# Crea un directorio y mira sus permisos por defecto
mkdir test_dir
ls -ld test_dir
