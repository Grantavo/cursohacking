Asignación de permisos [1-2]

Duración: 7:09 · Sección 2: Temario

Además de los permisos, cada archivo tiene un propietario y un grupo. Cambiar quién es dueño de un archivo es una tarea habitual en administración de sistemas.

chown — Cambiar propietario

El comando chown cambia el usuario propietario de un archivo o directorio. Solo root puede hacerlo.

chown usuario archivo.txt           # Cambia el propietario
chown usuario:grupo archivo.txt     # Cambia propietario y grupo a la vez
chown :grupo archivo.txt            # Cambia solo el grupo
chown -R usuario directorio/        # Cambia todo el contenido recursivamente

chgrp — Cambiar grupo

El comando chgrp cambia solo el grupo del archivo. Un usuario puede cambiar el grupo a cualquiera de los grupos a los que pertenece.

chgrp developers proyecto.txt       # Cambia el grupo a "developers"
chgrp -R staff /home/compartido/    # Cambia el grupo recursivamente
💡 chown usuario:grupo es equivalente a ejecutar chown usuario seguido de chgrp grupo.

Ejemplos con usuarios y grupos

# Crear usuarios y grupos de ejemplo
sudo useradd alice
sudo useradd bob
sudo groupadd developers

# Asignar propiedad
sudo chown alice:developers proyecto.txt
ls -l proyecto.txt
# -rw-r--r-- alice developers  123 May 10 12:00 proyecto.txt
1 Crear un directorio compartido 
sudo mkdir /home/compartido
sudo chown root:developers /home/compartido
sudo chmod 775 /home/compartido    # rwxrwxr-x
2 Transferir propiedad de un archivo 
sudo chown bob documento.txt
# Ahora bob es el dueño, puede cambiar sus permisos
⚠️ Solo root puede cambiar el propietario de un archivo. Un usuario normal no puede "regalar" sus archivos a otro usuario, ni siquiera siendo el dueño.

Verificación

Usa ls -l para ver propietario y grupo, o stat para información detallada:

stat archivo.txt
# File: archivo.txt
# Size: 123       Blocks: 8          ...
# Access: (0644/-rw-r--r--)  Uid: ( 1000/   alice)   Gid: ( 1001/developers)
