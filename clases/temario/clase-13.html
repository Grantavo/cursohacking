Asignación de permisos [2-2]

Duración: 6:31 · Sección 2: Temario

El modo octal es la forma más compacta y precisa de asignar permisos. Cada dígito representa los permisos de una categoría usando valores numéricos.

Valores de los permisos

PermisoValor
r (read)4
w (write)2
x (execute)1
- (ninguno)0

Los valores se suman para cada categoría (dueño, grupo, otros).

Combinaciones más comunes

OctalSimbólicoDescripción
777rwxrwxrwxTodos pueden hacer todo (inseguro)
755rwxr-xr-xDueño escribe, resto lee/ejecuta (típico de scripts)
700rwx------Solo el dueño accede
644rw-r--r--Dueño escribe, resto solo lee (típico de archivos)
600rw-------Solo dueño lee/escribe (archivos sensibles)
555r-xr-xr-xTodos leen/ejecutan, nadie escribe
400r--------Solo dueño lee

Cómo calcular permisos en octal

Para cada categoría suma los valores deseados:

# Si quieres: dueño rwx, grupo r-x, otros r-x
dueño:  r+w+x = 4+2+1 = 7
grupo:  r+_+x = 4+0+1 = 5
otros:  r+_+x = 4+0+1 = 5
Resultado: chmod 755 script.sh
🧮 Cálculo rápido:
  • 7 = rwx (todos los permisos)
  • 6 = rw- (leer y escribir)
  • 5 = r-x (leer y ejecutar)
  • 4 = r-- (solo lectura)
  • 0 = --- (ningún permiso)

Ejercicios prácticos

# Ejercicio 1: Script ejecutable (755)
chmod 755 script.sh
ls -l script.sh   # -rwxr-xr-x

# Ejercicio 2: Archivo privado (600)
chmod 600 id_rsa
ls -l id_rsa      # -rw-------

# Ejercicio 3: Directorio compartido (775)
chmod 775 /home/compartido
ls -ld /home/compartido  # drwxrwxr-x

# Ejercicio 4: Archivo público de solo lectura (444)
chmod 444 readme.txt
ls -l readme.txt  # -r--r--r--
Práctica Traduce de simbólico a octal 
rwxr-xr--  =  7 5 4  →  chmod 754
rwx------  =  7 0 0  →  chmod 700
rw-rw-r--  =  6 6 4  →  chmod 664
⚠️ Nunca pongas permisos 777 en producción. Es una mala práctica de seguridad. Si necesitas que varios usuarios accedan a algo, usa grupos y permisos 755 o 750.
