Notación octal de permisos

Duración: 7:21 · Sección 2: Temario

La notación octal es la forma numérica de representar permisos en Linux. Se llama "octal" porque cada dígito puede tomar valores del 0 al 7 (8 valores posibles, base 8).

El sistema octal explicado paso a paso

Cada permiso tiene un valor binario y su equivalente decimal:

BinarioPermisosOctal
000---0
001--x1
010-w-2
011-wx3
100r--4
101r-x5
110rw-6
111rwx7
💡 Piensa en cada trío de permisos como un número binario de 3 bits: r=4 (2²), w=2 (2¹), x=1 (2⁰). Activado = 1, desactivado = 0. El número octal es simplemente la suma.

Cálculos paso a paso

# Queremos: rwxr--r--
dueño: r(4)+w(2)+x(1) = 7
grupo: r(4)+0+0       = 4
otros: r(4)+0+0       = 4
→ chmod 744 archivo

# Queremos: rwxr-x---
dueño: r(4)+w(2)+x(1) = 7
grupo: r(4)+0+x(1)    = 5
otros: 0+0+0          = 0
→ chmod 750 archivo

# Queremos: rwx------
dueño: 4+2+1 = 7
grupo: 0
otros: 0
→ chmod 700 archivo

umask — Permisos por defecto

umask determina los permisos que se asignan por defecto al crear archivos y directorios. Funciona restando la máscara de los permisos base:

  • Archivos base: 666 (rw-rw-rw-)
  • Directorios base: 777 (rwxrwxrwx)
umask                     # Muestra la máscara actual (022 típicamente)
umask 022                 # Archivos: 666-022=644, Directorios: 777-022=755
umask 077                 # Archivos: 666-077=600, Directorios: 777-077=700

# Ejemplo práctico:
umask 077
touch secreto.txt
ls -l secreto.txt         # -rw-------  (600)
⚠️ umask afecta a la sesión actual. Para hacerlo permanente, añádelo a ~/.bashrc o ~/.profile.
