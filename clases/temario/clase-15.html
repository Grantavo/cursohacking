Permisos especiales - Sticky Bit

Duración: 5:09 · Sección 2: Temario

El Sticky Bit es un permiso especial que se aplica a directorios. Su función es evitar que un usuario borre archivos de otros usuarios aunque tenga permisos de escritura en el directorio.

¿Qué es el Sticky Bit?

Normalmente, si tienes permiso de escritura en un directorio, puedes eliminar cualquier archivo dentro de él, incluso si no eres el dueño. El Sticky Bit cambia este comportamiento: solo el dueño del archivo, el dueño del directorio o root pueden eliminar o renombrar archivos.

Ejemplo clásico: /tmp

$ ls -ld /tmp
drwxrwxrwt 1 root root 4096 May 10 12:00 /tmp

La letra t al final de los permisos indica que el Sticky Bit está activo. /tmp permite a todos escribir, pero gracias al Sticky Bit nadie puede borrar los archivos de otros.

💡 En la notación simbólica, el Sticky Bit se muestra como t en la posición de ejecución de "otros". Si también hay permiso de ejecución, aparece t en minúscula; si no hay ejecución, aparece T mayúscula.

Cómo se asigna

chmod +t directorio/               # Modo simbólico
chmod 1777 directorio/             # Modo octal (1 = Sticky Bit)
chmod o+t directorio/              # Alternativa simbólica

El Sticky Bit se representa con el dígito 1 al inicio del número octal:

chmod 1777 /tmp           # Permisos 777 + Sticky Bit
                          # 1 = Sticky Bit
                          # 7 = dueño rwx
                          # 7 = grupo rwx  
                          # 7 = otros rwx

Ejemplo práctico

# Crea un directorio compartido con Sticky Bit
sudo mkdir /home/compartido
sudo chmod 1777 /home/compartido
ls -ld /home/compartido    # drwxrwxrwt

# Ahora usuarios pueden crear archivos pero no borrar los de otros
su alice -c 'touch /home/compartido/alice.txt'
su bob -c 'rm /home/compartido/alice.txt'   # Error: permiso denegado
# Solo alice o root pueden borrar alice.txt
⚠️ El Sticky Bit no afecta a root. root siempre puede borrar cualquier archivo, independientemente del Sticky Bit.

¿Cuándo usar Sticky Bit?

  • Directorios compartidos (/tmp, /var/tmp)
  • Directorios de subida de archivos en servidores
  • Cualquier directorio donde múltiples usuarios tengan permiso de escritura
