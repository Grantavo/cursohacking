Permisos especiales - SUID y SGID

Duración: 7:09 · Sección 2: Temario

SUID (Set User ID) y SGID (Set Group ID) son permisos especiales que permiten ejecutar un programa con los privilegios del dueño o del grupo del archivo, en lugar de con los privilegios del usuario que lo ejecuta.

SUID — Set User ID

Cuando un archivo tiene SUID, se ejecuta con los permisos del dueño del archivo, no del usuario que lo ejecuta. En ls -l se ve como una s en la posición de ejecución del dueño.

$ ls -l /usr/bin/passwd
-rwsr-xr-x 1 root root 68208 May 10 12:00 /usr/bin/passwd
#       ^
#       s = SUID activo (dueño = root)
🔑 Ejemplo clásico: /usr/bin/passwd necesita modificar /etc/shadow, que solo root puede escribir. Gracias al SUID, cualquier usuario puede cambiar su propia contraseña ejecutando passwd con privilegios de root temporalmente.

SGID — Set Group ID

SGID tiene dos comportamientos:

  • En archivos: el programa se ejecuta con los permisos del grupo del archivo
  • En directorios: los nuevos archivos creados dentro heredan el grupo del directorio, no del usuario
# SGID en archivo (se ve como s en la posición de grupo)
ls -l /usr/bin/write
-rwxr-sr-x 1 root tty 18768 May 10 12:00 /usr/bin/write

# SGID en directorio
chmod g+s directorio/
ls -ld directorio/
drwxrws--- 2 alice developers 4096 May 10 12:00 directorio/
#       ^
#       s = SGID activo

Cómo se asigna

# SUID
chmod u+s programa           # Simbólico
chmod 4755 programa          # Octal (4 = SUID)

# SGID
chmod g+s directorio/        # Simbólico
chmod 2755 directorio/       # Octal (2 = SGID)
⚠️ Riesgos de seguridad: Un binario con SUID root es un objetivo para escalada de privilegios. Herramientas de enumeración como find / -perm -4000 2>/dev/null buscan estos binarios. Nunca pongas SUID en scripts o binarios que no sean absolutamente confiables.

Encontrar binarios con SUID/SGID

# Buscar archivos con SUID
find / -perm -4000 -type f 2>/dev/null

# Buscar archivos con SGID
find / -perm -2000 -type f 2>/dev/null

# Buscar ambos
find / \( -perm -4000 -o -perm -2000 \) -type f 2>/dev/null

Ejercicio práctico

# Consulta qué SUID tiene tu sistema
find / -perm -4000 -type f 2>/dev/null

# Observa los permisos de passwd
ls -l /usr/bin/passwd

# Crea un directorio con SGID para trabajo en grupo
sudo mkdir /home/grupo
sudo chown root:developers /home/grupo
sudo chmod 2775 /home/grupo    # rwxrwsr-x (SGID activo)
