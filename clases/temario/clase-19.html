Privilegios especiales - Capabilities

Duración: 5:00 · Sección 2: Temario

Las Linux Capabilities son una alternativa más granular y segura al SUID. En lugar de dar todos los privilegios de root, puedes otorgar capacidades específicas a un binario o proceso.

¿Por qué Capabilities?

SUID otorga privilegios completos de root, lo que es peligroso. Si un binario con SUID root tiene una vulnerabilidad, el atacante obtiene control total. Las capabilities dividen los privilegios de root en unidades atómicas: un binario solo tiene los permisos mínimos necesarios para funcionar.

Capabilities comunes

CapabilityDescripción
CAP_NET_RAWPermite crear sockets RAW (necesario para ping, nmap)
CAP_NET_ADMINPermite configurar la red (interfaces, firewall)
CAP_SYS_ADMINPermite operaciones administrativas (montar, swapon, etc.)
CAP_DAC_OVERRIDEPermite saltarse las comprobaciones de permisos rwx
CAP_SETUIDPermite cambiar el UID del proceso (como su, sudo)
CAP_KILLPermite enviar señales a cualquier proceso

Comandos: getcap y setcap

# Ver capabilities de un binario
getcap /usr/bin/ping
# /usr/bin/ping = cap_net_raw+ep

# Ver capabilities de todo el sistema
getcap -r / 2>/dev/null
# Asignar una capability a un binario
sudo setcap cap_net_raw+ep /usr/bin/mi_programa

# Quitar capabilities
sudo setcap -r /usr/bin/mi_programa

# El sufijo +ep significa:
#   e = Effective (la capability está activa)
#   p = Permitted (el binario puede usar esta capability)
💡 En sistemas modernos, /usr/bin/ping ya no usa SUID sino capabilities. Verifica: ls -l /usr/bin/ping (no tiene s) y getcap /usr/bin/ping (tiene cap_net_raw+ep).

SUID vs Capabilities

CaracterísticaSUIDCapabilities
PrivilegiosRoot completoSolo los necesarios
SeguridadBaja (todo o nada)Alta (mínimo privilegio)
Gestiónchmod u+ssetcap
Visibilidadls -l (letra s)getcap
ContenedoresNo recomendadoIdeal (Docker usa capabilities)
⚠️ No todas las capabilities son seguras. CAP_SYS_ADMIN es prácticamente equivalente a root. Siempre asigna la capability más específica posible.

Ejercicio práctico

# Inspeccionar ping en tu sistema
ls -l /usr/bin/ping
getcap /usr/bin/ping

# Buscar binarios con capabilities en el sistema
getcap -r /usr 2>/dev/null

# Asignar CAP_NET_RAW a un binario propio
sudo setcap cap_net_raw+ep ./mi_ping
getcap ./mi_ping
