Estructura de directorios del sistema

Duración: 7:14 · Sección 2: Temario

Linux organiza todo su sistema de archivos bajo una jerarquía estándar llamada FHS (Filesystem Hierarchy Standard). A diferencia de Windows, donde cada unidad tiene su propio árbol (C:\, D:\, etc.), en Linux todo parte desde la raíz /. Conocer esta estructura es fundamental para entender dónde encontrar archivos de configuración, programas, logs y dispositivos.

El árbol de directorios: de la raíz a las ramas

El directorio raíz / es el punto de partida. Dentro de él encontrarás estos directorios principales:

DirectorioContenido
/binBinarios esenciales del usuario (ls, cp, mv, cat)
/sbinBinarios del sistema (fdisk, mkfs, iptables)
/etcArchivos de configuración del sistema
/homeDirectorios personales de los usuarios
/varDatos variables: logs, bases de datos, colas de impresión
/tmpArchivos temporales (se borran al reiniciar)
/usrProgramas y librerías de usuario (binarios no esenciales)
/optSoftware opcional de terceros
/procSistema de archivos virtual con información de procesos
/devArchivos de dispositivo (discos, terminales, USB)

Inspeccionando directorios con ls -la

Ejecuta estos comandos para ver qué hay realmente dentro de cada directorio:

ls -la /            # Contenido de la raíz
ls -la /etc         # Archivos de configuración
ls -la /var/log     # Logs del sistema
ls -la /dev         # Dispositivos
💡 Tip: Fíjate en /etc/passwd (usuarios), /etc/shadow (contraseñas cifradas), /etc/apt/sources.list (repositorios de paquetes). Son archivos que revisarás constantemente en ciberseguridad.

Explorando /proc: el sistema de archivos virtual

/proc no existe realmente en el disco duro; el kernel lo crea en memoria para exponer información de procesos y hardware:

cat /proc/cpuinfo   # Información del procesador
cat /proc/meminfo   # Memoria RAM disponible y usada
cat /proc/version   # Versión del kernel
ls /proc/1/         # Información del proceso init (PID 1)

Dispositivos en /dev

En Linux, todo es un archivo. Los discos duros, las particiones, las terminales y los dispositivos USB aparecen como archivos dentro de /dev:

ls -la /dev/sda*    # Primer disco duro y sus particiones
ls -la /dev/tty*    # Terminales
ls -la /dev/null    # El agujero negro de Linux
ls -la /dev/random  # Generador de números aleatorios
⚠️ Cuidado: Los archivos en /dev representan dispositivos físicos. Escribir directamente en /dev/sda puede destruir datos del disco. No juegues con estos archivos sin saber lo que haces.

Entender esta jerarquía te permitirá navegar el sistema con confianza. Cuando un atacante compromete un servidor, lo primero que hace es inspeccionar /etc, /var/log y /home — saber qué esperar en cada directorio es clave tanto para ataque como para defensa.
