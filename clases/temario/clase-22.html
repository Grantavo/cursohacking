Actualización y Upgradeo del sistema

Duración: 3:41 · Sección 2: Temario

Mantener el sistema actualizado no solo te da las últimas funciones, sino que parchea vulnerabilidades de seguridad. En distribuciones basadas en Debian (como Ubuntu), el gestor de paquetes principal es apt. Entender la diferencia entre update, upgrade y cómo gestionar repositorios es esencial.

apt update vs apt upgrade vs apt full-upgrade

Estos tres comandos hacen cosas distintas y es importante no confundirlos:

ComandoQué hace
apt updateActualiza la lista de paquetes disponibles desde los repositorios
apt upgradeActualiza los paquetes instalados a la última versión (sin eliminar ni instalar nuevos)
apt full-upgradeActualiza todo, pudiendo instalar o eliminar dependencias si es necesario
# Flujo normal de actualización
sudo apt update
sudo apt upgrade -y

# Para una actualización completa
sudo apt update && sudo apt full-upgrade -y

Gestión de repositorios: /etc/apt/sources.list

Los repositorios son servidores desde donde apt descarga los paquetes. Se definen en /etc/apt/sources.list y en archivos dentro de /etc/apt/sources.list.d/:

# Ver el contenido de sources.list
cat /etc/apt/sources.list

# Ejemplo de entrada en Ubuntu
deb http://archive.ubuntu.com/ubuntu noble main restricted universe multiverse

# Añadir un repositorio (ej: Google Chrome)
echo "deb [arch=amd64] http://dl.google.com/linux/chrome/deb/ stable main" | sudo tee /etc/apt/sources.list.d/google-chrome.list
💡 Tip: Después de añadir un repositorio nuevo, siempre ejecuta sudo apt update para que apt reconozca los paquetes del nuevo repositorio.

dpkg: el gestor de paquetes de bajo nivel

Mientras que apt resuelve dependencias automáticamente, dpkg instala paquetes .deb de forma manual. Útil cuando descargas un archivo .deb directamente:

# Instalar un paquete .deb
sudo dpkg -i paquete.deb

# Si hay dependencias rotas, repararlas
sudo apt install -f

# Listar paquetes instalados
dpkg -l

# Saber a qué paquete pertenece un archivo
dpkg -S /bin/ls

Buscar paquetes con apt-cache

Antes de instalar, puedes buscar paquetes disponibles con apt-cache:

# Buscar paquetes que contengan "nmap"
apt-cache search nmap

# Mostrar información detallada de un paquete
apt-cache show nmap

# Ver dependencias de un paquete
apt-cache depends nmap
⚠️ Precaución: No hagas apt full-upgrade a mitad de un proyecto importante. A veces las actualizaciones rompen dependencias o cambian configuraciones. Siempre es buena práctica hacer una copia de seguridad antes de actualizar.

Dominar apt y dpkg es fundamental no solo para mantener tu sistema, sino también para instalar herramientas de hacking que muchas veces vienen en formato .deb o desde repositorios externos (como los de herramientas ofensivas).
