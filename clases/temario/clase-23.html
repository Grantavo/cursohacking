Uso y manejo con Tmux

Duración: 16:17 · Sección 2: Temario

Tmux (Terminal Multiplexer) te permite dividir tu terminal en múltiples paneles, crear ventanas y mantener sesiones activas aunque cierres la terminal. Es una herramienta indispensable para cualquier profesional de ciberseguridad: puedes dejar un escaneo de nmap corriendo en un panel, mientras editas un script en otro, y todo persistirá aunque pierdas la conexión SSH.

Conceptos básicos: sesiones, ventanas y paneles

Tmux tiene tres niveles de organización:

  • Sesión: Contenedor principal. Puedes tener varias sesiones para distintos proyectos.
  • Ventana: Dentro de una sesión, cada ventana es como una pestaña.
  • Panel: Cada ventana puede dividirse en múltiples paneles.

Crear y gestionar sesiones

# Crear una nueva sesión con nombre
tmux new -s mi-sesion

# Listar sesiones activas
tmux ls

# Adjuntarse a una sesión existente
tmux attach -t mi-sesion

# Matar una sesión
tmux kill-session -t mi-sesion

Comandos con el prefijo (Ctrl+B)

El prefijo por defecto es Ctrl+B. Después de pulsarlo, el siguiente comando se interpreta como una orden para Tmux:

CombinaciónAcción
Ctrl+B %Dividir panel verticalmente
Ctrl+B "Dividir panel horizontalmente
Ctrl+B cCrear nueva ventana
Ctrl+B nIr a la siguiente ventana
Ctrl+B pIr a la ventana anterior
Ctrl+B ,Renombrar la ventana actual
Ctrl+B dDesconectarse de la sesión (detach)
Ctrl+B flechasNavegar entre paneles
Ctrl+B xCerrar el panel actual

Personalización con .tmux.conf

Crea el archivo ~/.tmux.conf para cambiar el comportamiento por defecto. Aquí tienes una configuración recomendada:

# ~/.tmux.conf — Configuración personalizada

# Cambiar el prefijo a Ctrl+A (como screen)
set -g prefix C-a
unbind C-b
bind C-a send-prefix

# Dividir paneles más intuitivo
bind | split-window -h
bind - split-window -v

# Navegación con Alt + flechas
bind -n M-Left select-pane -L
bind -n M-Right select-pane -R
bind -n M-Up select-pane -U
bind -n M-Down select-pane -D

# Mouse habilitado
set -g mouse on

# Línea de estado con colores
set -g status-bg black
set -g status-fg white
set -g status-left '#[bg=green] #S '
set -g status-right '#[bg=red] %H:%M '
💡 Tip para hacking: Usa sesiones de Tmux con nombre para cada fase de un pentest: tmux new -s reconnaissance, tmux new -s exploitation, tmux new -s post-exploitation. Así mantienes todo organizado y puedes retomarlo al día siguiente.

Tmux en hacking: mantener sesiones activas

Imagina que estás haciendo fuerza bruta con Hydra desde una máquina remota por SSH. Sin Tmux, si tu conexión se cae, el proceso muere. Con Tmux:

# Conectarse por SSH y crear una sesión
ssh usuario@victima
tmux new -s brute-force

# Iniciar el ataque (tardará horas)
hydra -l admin -P rockyou.txt ssh://192.168.1.1

# Pulsar Ctrl+B d para desconectarte
# ¡El proceso sigue corriendo!
# Vuelve después, reconéctate por SSH y adjunta:
tmux attach -t brute-force
⚠️ Importante: Tmux no es mágico. Si reinicias la máquina remota, las sesiones se pierden. Tmux mantiene las sesiones mientras el sistema siga encendido, incluso si pierdes la conexión de red o cierras tu terminal local.

Una vez que empieces a usar Tmux, te preguntarás cómo trabajabas sin él. Es una de esas herramientas que parecen complejas al principio, pero que en una semana se vuelven parte de tu flujo natural.
