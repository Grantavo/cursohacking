Búsquedas a nivel de sistema

Duración: 11:03 · Sección 2: Temario

Encontrar archivos en Linux es una tarea diaria. Ya sea para localizar configuraciones, scripts maliciosos, logs antiguos o binarios específicos, saber usar find, locate, which y whereis te hará infinitamente más productivo. Cada comando tiene su propósito y sus ventajas.

find: el buscador definitivo

find recorre el sistema de archivos en tiempo real buscando archivos que cumplan condiciones. Es lento pero extremadamente flexible:

# Buscar por nombre
find / -name "shadow" 2>/dev/null

# Buscar por tipo (directorios)
find /var -type d -name "log" 2>/dev/null

# Buscar archivos mayores de 100MB
find / -type f -size +100M 2>/dev/null

# Buscar archivos modificados en los últimos 7 días
find /home -type f -mtime -7 2>/dev/null

# Buscar archivos con permisos 777 (inseguros)
find / -type f -perm 777 2>/dev/null
💡 ¿Por qué 2>/dev/null? find intenta acceder a directorios sin permisos y muestra errores. Redirigir stderr a /dev/null oculta esos mensajes y solo ves resultados útiles.

Ejecutar comandos con find y -exec

La opción -exec ejecuta un comando sobre cada archivo encontrado. El {} se reemplaza por el nombre del archivo y \; marca el final del comando:

# Encontrar archivos .conf y mostrar su contenido
find /etc -name "*.conf" -exec cat {} \;

# Buscar y eliminar archivos temporales antiguos
find /tmp -type f -atime +30 -exec rm {} \;

# Buscar y cambiar permisos
find /var/www -type f -exec chmod 644 {} \;

locate: la alternativa rápida

locate busca en una base de datos preconstruida, por lo que es instantáneo. La base de datos se actualiza con updatedb:

# Buscar instantáneamente
locate shadow
locate ".conf"
locate -i "README"   # -i ignora mayúsculas/minúsculas
⚠️ Limitación de locate: Si acabas de crear un archivo, locate no lo encontrará hasta que se ejecute updatedb (normalmente cada noche). Para archivos recién creados, usa find.

which, whereis y type

Estos comandos localizan programas ejecutables, no archivos arbitrarios:

# which: muestra la ruta completa del ejecutable
which nmap
which python3

# whereis: busca el binario, las fuentes y las páginas man
whereis ls
whereis ssh

# type: indica cómo se interpretaría un comando
type ls
type cd
type ll

type es especialmente útil para saber si un comando es un alias, una función, un builtin del shell o un ejecutable externo.

Ejemplos complejos con find

Combinando criterios puedes hacer búsquedas muy precisas. Aquí algunos escenarios reales:

# Archivos con setuid (riesgo de seguridad)
find / -type f -perm -4000 2>/dev/null

# Archivos de un usuario específico
find /home -type f -user nobody

# Archivos vacíos en /etc
find /etc -type f -empty

# Buscar por combinación de criterios (nombre Y tamaño)
find / -type f -name "*.log" -size +10M 2>/dev/null

# Buscar sockets de red abiertos como archivos
find / -type s 2>/dev/null

Saber buscar archivos es una habilidad que usarás cientos de veces. En ciberseguridad, encontrar archivos con permisos incorrectos, scripts sospechosos o configuraciones expuestas puede ser la diferencia entre detectar un ataque o pasar por alto una brecha.
