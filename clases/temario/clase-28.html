Conexiones SSH

Duración: 6:25 · Sección 2: Temario

SSH (Secure Shell) es el protocolo estándar para conectarse de forma segura a sistemas remotos. Como profesional de ciberseguridad, lo usarás a diario: para gestionar servidores, acceder a máquinas en un laboratorio de pentesting o, durante un engagement real, conectarte a sistemas comprometidos de forma controlada.

Conexión básica

La sintaxis fundamental de SSH es muy simple:

ssh usuario@192.168.1.100

# Si el servicio SSH corre en un puerto diferente (ej: 2222)
ssh usuario@192.168.1.100 -p 2222

# Conectarse con un usuario y hostname específicos
ssh -l root 10.10.10.10

La primera vez que te conectes a un servidor, SSH te mostrará la huella digital del servidor y te preguntará si confías en ella. Escribe yes para agregarla a ~/.ssh/known_hosts.

Autenticación con contraseña

Por defecto, SSH te pedirá la contraseña del usuario remoto. Es el método más simple, pero también el menos seguro para entornos de producción (sujeto a ataques de fuerza bruta).

ssh usuario@192.168.1.100
# Te pedirá: usuario@192.168.1.100's password:
# (escribe la contraseña y pulsa Enter)
⚠️ Seguridad: Nunca uses contraseñas débiles en SSH expuesto a Internet. Los bots escanean constantemente el puerto 22 probando combinaciones comunes. Usa autenticación por clave pública.

Autenticación con clave pública (ssh-keygen)

Genera un par de claves (pública y privada) en tu máquina local. La clave privada nunca se comparte; la pública se copia al servidor:

# Generar claves (por defecto RSA 3072 bits)
ssh-keygen -t rsa -b 4096 -C "tu-email@ejemplo.com"

# También puedes usar Ed25519 (más moderna y rápida)
ssh-keygen -t ed25519 -C "tu-email@ejemplo.com"

# Esto crea dos archivos:
# ~/.ssh/id_ed25519       → clave PRIVADA (nunca compartir)
# ~/.ssh/id_ed25519.pub   → clave PÚBLICA  (compartir)

Para copiar la clave pública al servidor remoto:

# Método fácil con ssh-copy-id
ssh-copy-id usuario@192.168.1.100

# Método manual (si no tienes ssh-copy-id)
cat ~/.ssh/id_ed25519.pub | ssh usuario@192.168.1.100 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

A partir de este momento, al conectarte SSH usará tu clave privada automáticamente y no te pedirá contraseña.

Archivo ~/.ssh/config: atajos para conexiones frecuentes

Si te conectas a muchos servidores, editar el archivo ~/.ssh/config te ahorrará escribir la IP y el usuario cada vez:

# ~/.ssh/config

Host lab
    HostName 192.168.1.100
    User usuario
    Port 22
    IdentityFile ~/.ssh/id_ed25519

Host servidor-prod
    HostName 203.0.113.50
    User admin
    Port 2222
    IdentityFile ~/.ssh/id_rsa_produccion

Host *
    ServerAliveInterval 60
    ServerAliveCountMax 3

Ahora puedes conectarte simplemente con:

ssh lab
ssh servidor-prod
💡 Tip de hacking: Durante un pentest, cuando encuentres credenciales SSH, copia la clave privada a tu máquina, asígnale permisos 600 y conéctate: chmod 600 clave_privada && ssh -i clave_privada usuario@host. Las claves privadas deben tener permisos restrictivos o SSH las rechazará.

SSH es mucho más que una shell remota. Con SSH puedes hacer túneles (forwarding de puertos), montar sistemas de archivos remotos con sshfs, transferir archivos con scp y rsync, e incluso usarlo como proxy SOCKS dinámico.
