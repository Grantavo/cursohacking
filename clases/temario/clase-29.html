Lectura de archivos especiales [1-2]

Duración: 5:47 · Sección 2: Temario

Linux expone información del kernel y del hardware a través de sistemas de archivos virtuales como /proc y /sys. Estos no son archivos reales en el disco duro; son interfaces que el kernel crea en memoria para que puedas consultar —y en algunos casos modificar— el estado del sistema en tiempo real. En ciberseguridad, leer estos archivos te da información valiosa sobre el sistema objetivo.

/proc: el sistema de archivos de procesos

Cada proceso en ejecución tiene un directorio en /proc con su PID como nombre. Dentro encuentras información detallada:

# Información del procesador
cat /proc/cpuinfo

# Memoria RAM
cat /proc/meminfo

# Versión del kernel y sistema operativo
cat /proc/version

# Tiempo de actividad del sistema (uptime)
cat /proc/uptime

# Sistemas de archivos montados
cat /proc/mounts

# Particiones y discos
cat /proc/partitions

Explorar /proc/net/

Dentro de /proc/net/ puedes ver información de red en tiempo real sin necesidad de herramientas externas:

# Conexiones TCP activas (IPv4)
cat /proc/net/tcp

# Conexiones TCP activas (IPv6)
cat /proc/net/tcp6

# Estadísticas de red
cat /proc/net/netstat

# Tabla ARP (mapeo IP ↔ MAC)
cat /proc/net/arp

# Interfaces de red
cat /proc/net/dev
💡 Tip: En /proc/net/tcp, las direcciones IP y puertos están en hexadecimal. Por ejemplo, 0100007F:0016 es 127.0.0.1:22. Aprende a leerlo o usa ss -tulanp que lo muestra ya formateado.

Más archivos útiles en /proc

# Argumentos del kernel al arrancar
cat /proc/cmdline

# Carga del sistema (load average)
cat /proc/loadavg

# Dispositivos DMA
cat /proc/dma

# Dispositivos PCI
cat /proc/pci

# Interrupciones
cat /proc/interrupts

# Información del sistema de archivos
cat /proc/filesystems

/sys: información de dispositivos y drivers

/sys expone información estructurada del kernel sobre dispositivos, controladores y buses:

# Dispositivos de bloque (discos)
ls /sys/block/

# Dispositivos USB conectados
ls /sys/bus/usb/devices/

# Información de la clase de red
ls /sys/class/net/
cat /sys/class/net/eth0/address   # MAC address
⚠️ Cuidado: Algunos archivos en /proc y /sys son de solo lectura, pero otros permiten escritura. Por ejemplo, /proc/sys/net/ipv4/ip_forward controla el forwarding de paquetes. Escribir valores incorrectos puede desestabilizar el sistema. Siempre lee primero con cat antes de modificar.

Leer /proc y /sys te da una visión profunda del sistema sin necesidad de instalar herramientas adicionales. En un pentest, estos archivos son una mina de oro para enumerar el sistema objetivo.
