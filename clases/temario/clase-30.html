Lectura de archivos especiales [2-2]

Duración: 2:38 · Sección 2: Temario

Continuamos explorando archivos especiales. En esta clase nos centramos en los dispositivos virtuales de /dev y en los archivos de información de procesos individuales en /proc/[PID]/. Dominar estos archivos es clave en la fase de enumeración de un pentest.

Dispositivos virtuales en /dev

Estos archivos no corresponden a hardware real, sino a funcionalidades del kernel accesibles como si fueran archivos:

ArchivoFunción
/dev/nullAgujero negro. Todo lo que se escribe aquí se descarta.
/dev/zeroProduce flujo continuo de bytes nulos (\\0).
/dev/randomGenera números aleatorios con entropía real (se bloquea si no hay suficiente).
/dev/urandomGenera números aleatorios sin bloqueo (usa PRNG).
# Descartar salida no deseada
comando_ruidoso 2>/dev/null

# Crear un archivo vacío de 1 MB (relleno con ceros)
dd if=/dev/zero of=archivo_vacio bs=1M count=1

# Generar 16 bytes aleatorios en hexadecimal
cat /dev/urandom | head -c 16 | xxd -p

# Limpiar un disco de forma segura (¡CUIDADO!)
# dd if=/dev/urandom of=/dev/sda bs=4M
⚠️ Peligro real: /dev/random se bloquea si el sistema no tiene suficiente entropía. En servidores sin ratón, teclado o discos SSD, puede agotarse. En esos casos usa /dev/urandom. La diferencia de seguridad es irrelevante para la mayoría de usos prácticos.

/proc/[PID]/ — Información por proceso

Cada proceso en ejecución tiene un directorio numerado con su PID. Allí puedes encontrar información detallada:

# Ver los PIDs activos
ls /proc/ | grep -E '^[0-9]+$'

# Información de un proceso específico (ej: PID 1 — systemd/init)
cat /proc/1/cmdline     # Comando exacto con el que se inició
cat /proc/1/environ     # Variables de entorno (separadas por \0)
ls -la /proc/1/fd/      # Descriptores de archivo abiertos
cat /proc/1/status      # Estado del proceso, memoria, privilegios
cat /proc/1/maps        # Mapa de memoria del proceso
💡 Uso en hacking: Leer /proc/[PID]/environ de un proceso puede revelar contraseñas o tokens en variables de entorno. Durante un pentest, lista los procesos con ps aux y examina los entornos de procesos interesantes (bases de datos, servidores web, etc.). 
# Script simple para extraer variables de entorno de todos los procesos (como root)
for pid in $(ls /proc/ | grep -E '^[0-9]+$'); do
    echo "=== PID $pid ==="
    cat /proc/$pid/environ 2>/dev/null | tr '\0' '\n'
done

Los descriptores de archivo en /proc/[PID]/fd/ son especialmente interesantes: son enlaces simbólicos a los archivos que el proceso tiene abiertos. Si un proceso mantiene abierto un archivo que fue eliminado, aún puedes leer su contenido a través de este enlace, lo que puede ser útil en análisis forense.
