Directorios y archivos ocultos

Duración: 3:47 · Sección 2: Temario

En Linux, un archivo o directorio se considera "oculto" simplemente porque su nombre empieza con un punto (.). No es una característica de seguridad, sino una conveniencia para mantener ordenado el directorio personal. Sin embargo, en ciberseguridad, los archivos ocultos son cruciales: allí se esconden configuraciones sensibles, claves SSH y, a veces, malware.

¿Qué son los archivos ocultos?

Archivos como .bashrc, .gitconfig o .ssh son ocultos. El comando ls sin opciones no los muestra. Para verlos:

# ls normal — no muestra archivos ocultos
ls ~/

# ls -a — muestra TODOS los archivos, incluidos ocultos
ls -a ~/

# ls -A — como -a pero omite . y ..
ls -A ~/

# También puedes listar solo los ocultos
ls -d ~/.* 2>/dev/null

Archivos de configuración ocultos importantes en ~

Cada programa de terminal suele guardar su configuración en un archivo oculto en tu home:

ArchivoPrograma
~/.bashrcConfiguración de Bash
~/.zshrcConfiguración de Zsh
~/.vimrcConfiguración de Vim
~/.tmux.confConfiguración de Tmux
~/.gitconfigConfiguración global de Git
~/.ssh/Claves SSH y configuración
~/.config/Directorio estándar para configs de muchas apps modernas

El directorio .ssh

~/.ssh/ merece mención especial por su importancia en seguridad:

# Estructura típica de ~/.ssh/
ls -la ~/.ssh/

# Archivos comunes:
# id_rsa / id_ed25519       → claves privadas (NUNCA compartir)
# id_rsa.pub               → clave pública
# authorized_keys           → claves públicas permitidas para conectarse
# known_hosts               → huellas de servidores conocidos
# config                    → atajos de conexión
⚠️ En un pentest: Si encuentras un archivo id_rsa (clave privada SSH) en un sistema comprometido, puedes usarlo para conectarte a otros sistemas. Es uno de los hallazgos más valiosos durante la escalada de privilegios y movimiento lateral.

Crear archivos y directorios ocultos

Es tan simple como empezar el nombre con un punto:

# Crear un archivo oculto
touch .secreto.txt
echo "información sensible" > .secreto.txt

# Crear un directorio oculto
mkdir .mi_directorio_oculto

# Mover un archivo existente a oculto
mv configuracion.txt .configuracion.txt

# Visibilidad con ls -la
ls -la
💡 Dato curioso: Los directorios . (directorio actual) y .. (directorio padre) también son archivos ocultos. Siempre existen en cualquier directorio de Linux. Por eso ls -a siempre muestra al menos estos dos.

¿Por qué se usan archivos ocultos?

La razón es mantener el directorio home limpio visualmente. Imagina que cada programa dejara su archivo de configuración visible: tendrías decenas de archivos saturando tu ls. Al usar el punto, esos archivos quedan fuera de la vista diaria pero accesibles cuando los necesitas. En ciberseguridad, los atacantes a menudo esconden herramientas y scripts en directorios con nombre de punto (ej: .cache/, .dbus/) para pasar desapercibidos.
