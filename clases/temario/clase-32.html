Detección del tipo y formato de archivos

Duración: 2:49 · Sección 2: Temario

En Linux, la extensión de un archivo (.txt, .pdf, .exe) no determina su tipo real. Para saber qué contiene realmente un archivo se usa el comando file, que analiza los bytes mágicos (magic numbers) al inicio del archivo. Esto es esencial en ciberseguridad: un atacante puede renombrar un ejecutable malicioso como documento.pdf, pero file revelará su verdadera naturaleza.

El comando file

# Uso básico
file archivo

# Ejemplo con varios tipos
file /bin/ls
file /etc/passwd
file imagen.png
file documento.pdf

# Ver todos los tipos MIME (más detallado)
file -i /bin/ls
file -i imagen.png

Ejemplos prácticos

# file detecta el formato real sin importar la extensión
mv /bin/ls /tmp/falso.txt
file /tmp/falso.txt
# Output: falso.txt: ELF 64-bit LSB executable, x86-64, version 1 (SYSV)

# Un script de Bash sigue siendo texto aunque lo renombres
echo "#!/bin/bash\necho hola" > script.renombrado
file script.renombrado
# Output: script.renombrado: Bourne-Again shell script, ASCII text executable

Entendiendo los magic numbers

Los primeros bytes de un archivo identifican su formato. Puedes verlos con xxd u od:

# Ver los primeros bytes (magic number) de un archivo
xxd -l 16 /bin/ls | head -1
# Output: 00000000: 7f45 4c46 02...  → ELF

# Comparar con un PNG
xxd -l 16 imagen.png | head -1
# Output: 00000000: 8950 4e47 0d0a 1a0a  → PNG
FormatoMagic Number (hex)
ELF (binario Linux)7f 45 4c 46
PNG (imagen)89 50 4e 47
JPEGff d8 ff e0
PDF25 50 44 46
ZIP50 4b 03 04
💡 Aplicación en hacking: Durante un análisis de malware o un CTF, usa file para identificar rápidamente el tipo de archivos sospechosos. Un archivo que parece un .txt pero resulta ser un ELF o un ZIP puede ser la pista que necesitas.

La base de datos de tipos que usa file está en /usr/share/misc/magic o /usr/share/file/magic/. Puedes consultarla para entender cómo se identifican los formatos.
