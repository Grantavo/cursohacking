Búsquedas precisas de archivos [1-2]

Duración: 3:05 · Sección 2: Temario

Ya vimos find en la clase 24, pero ahora vamos a profundizar en los criterios de búsqueda y cómo combinarlos con operadores lógicos. Hacer búsquedas precisas te permite encontrar exactamente lo que necesitas sin tener que revisar cientos de resultados.

Criterios de búsqueda en find

CriterioDescripciónEjemplo
-nameBusca por nombre exactofind / -name "passwd"
-inameBusca por nombre (ignora mayúsculas)find / -iname "PassWd"
-type fSolo archivos regularesfind / -type f -name "*.txt"
-type dSolo directoriosfind / -type d -name "log"
-sizePor tamaño (+100M, -1k, 1024c)find / -size +1G
-mtimePor fecha de modificación (días)find / -mtime -7
-permPor permisosfind / -perm 644
-userPor propietariofind / -user root
-groupPor grupofind / -group www-data

Operadores lógicos: -a (and), -o (or), -not

# AND implícito: todos los criterios se cumplen
find / -type f -name "*.log" -size +50M

# AND explícito
find / -type f -a -name "*.conf"

# OR: archivos que se llamen access.log O error.log
find /var/log -name "access.log" -o -name "error.log"

# NOT: archivos que NO sean .txt
find /home -type f -not -name "*.txt"

# Combinaciones complejas
find / \( -name "*.py" -o -name "*.sh" \) -type f -size -1M
💡 Los paréntesis: En find, los paréntesis se escapan con \( y \) porque tienen significado especial para el shell. Sirven para agrupar condiciones lógicas.

Ejemplos avanzados

# Archivos con permisos SUID (riesgo de escalada de privilegios)
find / -type f -perm -4000 2>/dev/null

# Archivos modificados en la última hora (para detectar actividad reciente)
find /home -type f -mmin -60 2>/dev/null

# Directorios con permisos 777 (cualquiera puede escribir)
find / -type d -perm 777 2>/dev/null

# Archivos de más de 100MB que no sean .iso
find / -type f -size +100M -not -name "*.iso" 2>/dev/null

# Buscar archivos de un usuario específico en /tmp
find /tmp -user nobody -type f 2>/dev/null

Dominar estos criterios y operadores te permite construir búsquedas quirúrgicas. En un pentest, usar find con -perm -4000 es una de las primeras cosas que haces para buscar vectores de escalada de privilegios.
