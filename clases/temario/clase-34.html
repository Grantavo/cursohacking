Búsquedas precisas de archivos [2-2]

Duración: 1:53 · Sección 2: Temario

Cerramos el tema de búsquedas comparando find y locate, y viendo cómo ejecutar acciones sobre los archivos encontrados con -exec y xargs. Saber cuándo usar cada herramienta te ahorrará tiempo y recursos.

locate: búsqueda instantánea (pero no siempre actualizada)

locate busca en una base de datos preconstruida (/var/lib/mlocate/mlocate.db), actualizada periódicamente por updatedb (normalmente cada noche via cron).

# Buscar instantáneamente
locate shadow
locate "*.conf"
locate -i "readme"     # ignore-case

# Contar resultados
locate -c "*.log"

# Limitar número de resultados
locate -l 10 "*.log"

# Forzar actualización de la base de datos
sudo updatedb
💡 updatedb manual: Ejecuta sudo updatedb después de instalar o crear archivos si quieres que locate los encuentre de inmediato. En servidores de pentesting, puede ser útil actualizar la base antes de buscar.

find con -exec y xargs

Cuando necesitas hacer algo con los archivos encontrados, tienes dos opciones:

# -exec: ejecuta un comando por cada archivo
find /tmp -type f -name "*.tmp" -exec rm {} \;

# xargs: pasa los archivos como argumentos a un comando (más eficiente)
find /tmp -type f -name "*.tmp" | xargs rm -f

# Diferencia clave: -exec ejecuta el comando una vez por archivo
# mientras que xargs agrupa varios archivos en una sola ejecución

Ejemplos prácticos con -exec y xargs

# Cambiar permisos recursivamente
find /var/www -type f -exec chmod 644 {} \;
find /var/www -type d -exec chmod 755 {} \;

# Buscar y copiar archivos de log a un directorio de backup
find /var/log -name "*.log" -mtime -1 -exec cp {} /backup/logs/ \;

# Buscar archivos que contengan una palabra (combinando find + grep)
find /etc -type f -name "*.conf" -exec grep -l "PermitRootLogin" {} \;

# Usando xargs para borrar archivos antiguos
find /tmp -type f -atime +7 -print | xargs rm -f

# Buscar archivos por contenido (sin -exec)
find /var -type f -name "*.php" -exec grep -l "password" {} \;

Diferencias clave entre find y locate

Aspectofindlocate
VelocidadLenta (recorre el sistema)Instantánea (base de datos)
ActualizaciónTiempo realDepende de updatedb
FiltrosMuy flexibles (nombre, tamaño, fecha, permisos...)Solo por nombre/patrón
-exec/xargsSí, integradoNo directamente
PermisosSolo ve lo que el usuario puede leerPuede listar archivos que el usuario no puede leer si la DB es global
⚠️ Seguridad con locate: La base de datos de locate puede contener rutas de archivos que el usuario no debería conocer (porque no tiene permisos de lectura). Por eso, en sistemas multiusuario, la DB de locate suele actualizarse con privilegios limitados.

En la práctica: usa locate para búsquedas rápidas de archivos que sabes que existen (configuraciones, binarios comunes). Usa find cuando necesites criterios precisos (tamaño, fecha, permisos) o cuando el archivo pueda ser muy reciente.
