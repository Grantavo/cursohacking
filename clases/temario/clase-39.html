Cifrado César y uso de tr para la traducción de caracteres

Duración: 4:42 · Sección 2: Temario

El cifrado César es uno de los métodos criptográficos más antiguos: desplaza cada letra del alfabeto un número fijo de posiciones. En Linux, el comando tr (translate) es la herramienta ideal para implementarlo.

El comando tr

tr sustituye, elimina o comprime caracteres de la entrada estándar. Su sintaxis es tr 'conjunto_origen' 'conjunto_destino'.

# Convertir minúsculas a mayúsculas
echo "hola mundo" | tr 'a-z' 'A-Z'

# Cifrado César con desplazamiento +3
echo "hola" | tr 'a-z' 'd-za-c'

# Descifrar (desplazamiento -3)
echo "krod" | tr 'a-z' 'x-za-w'

# ROT13 (desplazamiento +13, simétrico)
echo "Hola Mundo" | tr 'A-Za-z' 'N-ZA-Mn-za-m'
🧠 ROT13: Es un César con desplazamiento 13. Como el alfabeto tiene 26 letras, aplicar ROT13 dos veces devuelve el texto original. Es el cifrado César más usado en entornos UNIX (por ejemplo, para ocultar spoilers en Usenet).

Operaciones avanzadas con tr

Además de sustitución, tr permite eliminar y comprimir caracteres con las opciones -d y -s.

# Eliminar caracteres específicos
echo "Hola, mundo!!!" | tr -d '!,'

# Comprimir espacios múltiples
echo "Hola     mundo" | tr -s ' '

# Eliminar dígitos
echo "Clave: 12345" | tr -d '0-9'

# Combinar: pasar a mayúsculas y eliminar puntuación
echo "Hola, mundo!" | tr 'a-z' 'A-Z' | tr -d '[:punct:]'

Fuerza bruta de César con bash

Podemos automatizar la fuerza bruta de un cifrado César usando un bucle for y tr:

#!/bin/bash
texto_cifrado="krod"
for i in {1..25}; do
  echo "Desplazamiento $i: $(echo "$texto_cifrado" | tr 'a-z' "$(echo {a..z} | tr -d ' ' | sed "s/^\(.\{$i\}\)\(.*\)/\2\1/")")"
done
⚠️ Nota de seguridad: El cifrado César no proporciona seguridad real. Se rompe en segundos con análisis de frecuencias. Sin embargo, es excelente para aprender conceptos de criptografía, pipes y manipulación de texto en la terminal.
