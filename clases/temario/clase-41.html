Manejo de pares de claves y conexiones SSH

Duración: 8:30 · Sección 2: Temario

SSH (Secure Shell) es el protocolo estándar para administrar servidores remotos de forma segura. En lugar de usar contraseñas, lo más seguro y conveniente es usar un par de claves pública-privada.

Generación de claves con ssh-keygen

El primer paso es generar un par de claves RSA de 4096 bits en tu máquina local.

# Generar par de claves RSA
ssh-keygen -t rsa -b 4096 -C "mi_correo@ejemplo.com"

# Los archivos se guardan por defecto en:
# ~/.ssh/id_rsa       (clave privada - NUNCA compartir)
# ~/.ssh/id_rsa.pub   (clave pública - se copia al servidor)

# Verificar la huella digital
ssh-keygen -lf ~/.ssh/id_rsa.pub
🔑 Anatomía de una clave pública: ssh-rsa AAAAB3NzaC1yc2E... usuario@host. El primer campo es el tipo de clave, el segundo es la clave en base64, y el tercero es un comentario (normalmente usuario@hostname).

Copiar la clave al servidor

El comando ssh-copy-id hace todo el trabajo: crea el directorio ~/.ssh en el servidor, añade tu clave a ~/.ssh/authorized_keys y establece los permisos correctos.

# Copiar clave pública al servidor remoto
ssh-copy-id -i ~/.ssh/id_rsa.pub usuario@192.168.1.100

# Manualmente (si no tienes ssh-copy-id)
cat ~/.ssh/id_rsa.pub | ssh usuario@192.168.1.100 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"

# Conectar sin contraseña
ssh usuario@192.168.1.100

ssh-agent y ssh-add

Si tu clave privada tiene passphrase (recomendado), ssh-agent la almacena en memoria para no tener que escribirla cada vez.

# Iniciar el agente
eval "$(ssh-agent -s)"

# Añadir la clave privada
ssh-add ~/.ssh/id_rsa

# Listar claves cargadas
ssh-add -l

# Eliminar todas las claves
ssh-add -D

Configuración avanzada con ~/.ssh/config

El archivo de configuración permite definir alias y opciones por host, ahorrando mucho tiempo en el día a día.

# ~/.ssh/config
Host servidor-prod
    HostName 192.168.1.100
    Port 2222
    User admin
    IdentityFile ~/.ssh/id_rsa_prod

Host github.com
    User git
    IdentityFile ~/.ssh/id_rsa_github

# Ahora conectas con:
ssh servidor-prod
⚠️ Seguridad: La clave privada (id_rsa) debe tener permisos 600 (solo lectura para el propietario). El directorio ~/.ssh/ debe tener permisos 700. Si los permisos son incorrectos, SSH se negará a usar las claves.
