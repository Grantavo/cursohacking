Detección de diferencias entre archivos

Duración: 2:49 · Sección 2: Temario

Comparar archivos es una tarea frecuente en administración y hacking: detectar cambios en configuraciones, verificar integridad de backups, o analizar diferencias entre versiones de un binario. Linux ofrece varias herramientas para ello.

diff: el estándar

diff compara archivos línea por línea y muestra las diferencias en varios formatos.

# Comparación básica
diff archivo1.txt archivo2.txt

# Formato unificado (más legible)
diff -u archivo1.txt archivo2.txt

# Formato contextual
diff -c archivo1.txt archivo2.txt

# Ignorar espacios y mayúsculas
diff -bi archivo1.txt archivo2.txt

# Comparar recursivamente directorios
diff -rq dir1/ dir2/
📖 Formato unificado (-u): Muestra 3 líneas de contexto antes y después de cada cambio. Las líneas eliminadas empiezan con -, las añadidas con +, y el contexto con espacio. Es el formato estándar para parches y la mayoría de herramientas modernas.

sdiff y cmp

sdiff muestra las diferencias lado a lado, mientras que cmp compara byte a byte (ideal para binarios).

# Comparación lado a lado
sdiff archivo1.txt archivo2.txt

# Comparación byte a byte (binarios)
cmp -l binario1.bin binario2.bin

# Saber si dos archivos son idénticos (solo éxito/fallo)
cmp -s archivo1 archivo2 && echo "Iguales" || echo "Diferentes"

Parches con diff y patch

El flujo diff + patch permite crear parches que aplican cambios de un archivo a otro. Muy usado en desarrollo de software y CTFs.

# Crear un parche
diff -u original.txt modificado.txt > cambios.patch

# Aplicar el parche
patch < cambios.patch

# Revertir el parche
patch -R < cambios.patch

# Aplicar a un archivo específico
patch archivo.txt < cambios.patch
⚠️ Aplicación en hacking: En un CTF, puedes usar diff para comparar configuraciones originales con las del sistema comprometido y detectar backdoors. Por ejemplo, comparar /etc/passwd original con el actual para encontrar usuarios añadidos: diff -u passwd_original passwd_actual.
