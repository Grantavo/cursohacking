Ejecución de comandos por SSH

Duración: 2:17 · Sección 2: Temario

Una de las capacidades más potentes de SSH es ejecutar comandos remotos sin abrir una sesión interactiva. Esto permite automatizar tareas de administración y auditoría de forma sencilla y segura.

Ejecución remota directa

Basta con pasar el comando como argumento a SSH. Se ejecuta en el servidor remoto y la salida se muestra en tu terminal local.

# Ejecutar un comando simple
ssh usuario@192.168.1.100 "whoami"

# Ejecutar múltiples comandos
ssh usuario@192.168.1.100 "uptime && free -h && df -h"

# Comando con pipes
ssh usuario@192.168.1.100 "ps aux | grep apache | wc -l"

# Redirigir salida local (el > se ejecuta LOCAL)
ssh usuario@192.168.1.100 "cat /etc/passwd" > passwd_remoto.txt
🧠 Comillas simples vs dobles: Con comillas dobles ("), las variables ($HOME) se expanden localmente antes de enviarse. Con comillas simples ('), las variables se expanden en el servidor remoto. Ejemplo: ssh host "echo $HOSTNAME" muestra el hostname local; ssh host 'echo $HOSTNAME' muestra el remoto.

Transferencia y ejecución de scripts

Podemos ejecutar scripts locales en el servidor remoto sin copiarlos explícitamente, usando redirección de entrada.

# Ejecutar un script local en remoto
ssh usuario@192.168.1.100 < mi_script.sh

# Con argumentos para el script
ssh usuario@192.168.1.100 'bash -s' < mi_script.sh --arg1 --arg2

# Copiar y ejecutar en un paso
scp mi_script.sh usuario@192.168.1.100:/tmp/ && ssh usuario@192.168.1.100 "/tmp/mi_script.sh"

Pseudo-terminal con -t

A veces necesitas una pseudo-terminal (PTY) incluso para comandos individuales, por ejemplo para comandos que requieren interacción o que verifican isatty().

# Forzar pseudo-terminal
ssh -t usuario@192.168.1.100 "sudo systemctl restart nginx"

# Comandos interactivos (htop, nano...)
ssh -t usuario@192.168.1.100 "sudo htop"

# Múltiples -t para forzar incluso sin TTY local
ssh -tt usuario@192.168.1.100 "comando"

# Ejecutar y mantener sesión
ssh -t usuario@192.168.1.100 "comando; exec bash -l"
⚠️ Automatización segura: Para automatizar tareas por SSH, usa siempre claves SSH sin passphrase (o con ssh-agent). Nunca incrustes contraseñas en scripts. Restringe los comandos que la clave puede ejecutar desde ~/.ssh/authorized_keys usando command="..." al inicio de la línea de la clave pública.
