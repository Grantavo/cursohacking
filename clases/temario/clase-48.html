Jugando con conexiones

Duración: 4:03 · Sección 2: Temario

Bash tiene capacidades de red nativas a traves de los pseudo-dispositivos /dev/tcp y /dev/udp. Esto permite hacer conexiones sin necesidad de netcat, curl o wget. Es especialmente util en sistemas minimalistas o cuando estas limitado por firewalls que solo permiten ciertos binarios.

/dev/tcp y /dev/udp en Bash

Bash puede abrir sockets TCP/UDP usando la sintaxis de redireccion. Esto funciona solo en Bash compilado con soporte de red (--enable-net-redirections), que es el caso en la mayoria de distribuciones.

# Abrir una conexion TCP y leer el banner
exec 3<>/dev/tcp/192.168.1.1/22
cat <&3

# Enviar datos y recibir respuesta
exec 3<>/dev/tcp/example.com/80
echo -e "GET / HTTP/1.1\r\nHost: example.com\r\nConnection: close\r\n\r\n" >&3
cat <&3
exec 3>&-
Explicacion de exec 3<>/dev/tcp: exec 3<>.../dev/tcp/IP/PORT asigna el descriptor de archivo 3 (puede ser cualquier numero) para lectura y escritura sobre la conexion TCP. Luego se lee con cat <& 3 y se escribe con echo ... >&3. Se cierra con exec 3>&-.

Enviar peticiones HTTP con Bash nativo

Podemos construir un cliente HTTP minimalista completamente en Bash, sin herramientas externas.

#!/bin/bash
host="$1"
ruta="${2:-/}"

exec 3<>/dev/tcp/$host/80
echo -e "GET $ruta HTTP/1.1\r\nHost: $host\r\nConnection: close\r\n\r\n" >&3
cat <&3
exec 3>&-

Download de archivos con Bash nativo

Combinando /dev/tcp con redirecciones, podemos descargar archivos sin curl ni wget.

#!/bin/bash
# Descargar un archivo usando solo Bash
download() {
    local host="$1"
    local ruta="$2"
    local salida="$3"

    exec 3<>/dev/tcp/$host/80
    echo -e "GET $ruta HTTP/1.1\r\nHost: $host\r\nConnection: close\r\n\r\n" >&3

    # Saltar cabeceras HTTP y guardar el cuerpo
    while IFS= read -r linea; do
        [ "$linea" = $'\r' ] && break
    done <&3

    cat <&3 > "$salida"
    exec 3>&-
}

download "example.com" "/archivo.zip" "descarga.zip"
Limitacion importante: El pseudo-dispositivo /dev/tcp solo funciona en Bash, no en sh, dash ni zsh. Verifica que tu shell sea Bash con echo $SHELL o bash --version. En sistemas embedded puede no estar compilado con soporte de red.
