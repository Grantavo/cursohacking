Abusando de tareas Cron [1-3]

Duración: 4:36 · Sección 2: Temario

Cron es el administrador de tareas programadas en Linux. Ejecuta comandos en intervalos definidos. Cuando estan mal configuradas, las tareas Cron son una via comun de escalada de privilegios en pentesting.

Que es Cron?

Cron lee archivos de configuracion llamados crontabs y ejecuta los comandos en el momento especificado. Cada usuario puede tener su propio crontab, y el sistema tiene crontabs globales.

# Editar el crontab del usuario actual
crontab -e

# Listar tareas programadas del usuario
crontab -l

# Formato del crontab:
# minuto hora dia mes dia_semana comando
# 0 5 * * * /usr/bin/backup.sh

# Ver el crontab de otro usuario (como root)
crontab -u usuario -l
Formato cron: Cada linea tiene 5 campos de tiempo seguidos del comando: minuto (0-59), hora (0-23), dia del mes (1-31), mes (1-12), dia de la semana (0-7, donde 0 y 7 son domingo). Un asterisco * significa "cualquier valor".

Directorios Cron del sistema

Linux tiene directorios especiales donde los scripts se ejecutan automaticamente segun su frecuencia. No requieren editar crontabs.

# Directorios cron del sistema
ls -la /etc/cron.hourly/   # Cada hora
ls -la /etc/cron.daily/    # Cada dia
ls -la /etc/cron.weekly/   # Cada semana
ls -la /etc/cron.monthly/  # Cada mes

# Archivo principal del sistema
cat /etc/crontab

Identificar tareas Cron vulnerables

Como atacante, buscamos scripts ejecutados por root que sean modificables por nuestro usuario. Un script escrito por root pero editable por cualquiera es una escalada de privilegios inmediata.

# Listar procesos y buscar tareas cron activas
ps aux | grep cron

# Ver el log de cron para ver que se ejecuta
grep CRON /var/log/syslog

# Buscar scripts ejecutados por root con permisos debiles
find /etc/cron* -type f -writable 2>/dev/null

# Verificar el propietario y permisos de scripts en cron
ls -la /etc/cron.d/
cat /etc/cron.d/*
Vector de ataque tipico: Si encuentras un script en /etc/cron.d/backup.sh propiedad de root pero con permisos 777 o 766, puedes sobrescribirlo con tu propio comando malicioso. En la siguiente ejecucion de Cron, tu comando se ejecutara como root.
