Comprendiendo las expresiones de las tareas Cron

Duración: 16:53 · Seccion 2: Temario

La sintaxis de Cron puede parecer críptica al principio, pero una vez que entiendes sus componentes, puedes expresar cualquier programacion temporal imaginable. Esta clase cubre desde lo basico hasta expresiones complejas, atajos y herramientas.

Sintaxis completa de Cron

Cada linea de un crontab tiene seis campos separados por espacios (o tabuladores): minuto, hora, dia del mes, mes, dia de la semana y comando.

# Formato: MIN HORA DIA MES DIA_SEMANA COMANDO
# Ejemplo: ejecutar a las 5:30am cada dia
30 5 * * * /usr/bin/backup.sh

# Explicacion:
# MIN:   0-59   (30)
# HORA:  0-23   (5)
# DIA:   1-31   (* = cualquier dia)
# MES:   1-12   (* = cualquier mes)
# DIA_SEM: 0-7  (* = cualquier dia, 0 y 7 = domingo)
Dia de la semana: 0 y 7 son domingo, 1 es lunes, ..., 6 es sabado. Puedes usar abreviaturas en ingles: SUN, MON, TUE, WED, THU, FRI, SAT (solo en algunos sistemas).

Expresiones complejas

Cron soporta operadores especiales para expresar patrones mas complejos:

Operador Significado Ejemplo Explicacion
* Todos los valores * * * * * Cada minuto
*/n Cada n unidades */5 * * * * Cada 5 minutos
a-b Rango 9-17 * * * * Cada hora de 9am a 5pm
a,b,c Lista de valores 0 9,12,15 * * * A las 9am, 12pm y 3pm
a-b/n Rango con paso 0 8-18/2 * * * Cada 2 horas entre 8am y 6pm 
# Ejemplos practicos de cada combinacion

# Cada 15 minutos
*/15 * * * * /scripts/check.sh

# Cada hora en punto (minuto 0)
0 * * * * /scripts/hourly.sh

# Primer dia de cada mes a las 2am
0 2 1 * * /scripts/monthly.sh

# Cada dia laborable (lunes a viernes) a las 8am
0 8 * * 1-5 /scripts/workday.sh

# Cada 30 minutos en horario laboral (9-18)
*/30 9-18 * * 1-5 /scripts/check_work.sh

# Cada 10 minutos los fines de semana
*/10 * * * 0,6 /scripts/weekend.sh

Atajos especiales (systemd/cron)

Ademas de la sintaxis clasica, muchos sistemas soportan atajos legibles en /etc/crontab y en archivos bajo /etc/cron.d/:

# Atajos de cron
@reboot     Ejecutar al arrancar el sistema
@yearly     Equivale a: 0 0 1 1 *
@annually   Equivale a: 0 0 1 1 *
@monthly    Equivale a: 0 0 1 * *
@weekly     Equivale a: 0 0 * * 0
@daily      Equivale a: 0 0 * * *
@hourly     Equivale a: 0 * * * *

# Ejemplos con atajos
@reboot /usr/local/bin/restore_iptables.sh
@daily /usr/local/bin/logrotate.sh
@hourly /usr/local/bin/check_temperature.sh

Herramientas utiles

crontab.guru es un sitio web que traduce expresiones cron a lenguaje natural. Introduce */15 9-17 * * 1-5 y te dira "At every 15th minute past every hour from 9 through 17 on every day-of-week from Monday through Friday".

# Verificar cuando se ejecutara una expresion cron
# Podemos usar cronologia o calcularlo manualmente:

# Ejemplo: */30 8-18 * * 1-5
# Se ejecutara: cada 30 minutos, entre 8am y 6pm,
# de lunes a viernes

# Proximo calculo mental:
# 8:00, 8:30, 9:00, 9:30, ..., 18:00
# Solo si el dia es lun-vie
Tip: Cuando disenies expresiones cron complejas, usa crontab.guru para verificar que tu expresion hace lo que crees. Es especialmente util con rangos combinados con pasos, donde es facil equivocarse.
