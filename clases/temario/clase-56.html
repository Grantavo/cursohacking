Operando con proyectos de Github [1-5]

Duración: 2:31 · Sección 2: Temario

Git es un sistema de control de versiones distribuido. GitHub es la plataforma más grande donde se alojan repositorios Git. En esta primera parte aprenderás a clonar y explorar repositorios.

Clonar un repositorio

Con git clone descargas una copia completa del repositorio remoto a tu máquina local:

# Clonar con HTTPS (recomendado para principiantes)
git clone https://github.com/usuario/repositorio.git

# Clonar con SSH (requiere clave SSH configurada)
git clone git@github.com:usuario/repositorio.git

# Clonar una rama específica
git clone -b nombre-rama https://github.com/usuario/repositorio.git

Navegar por el repositorio

Una vez clonado, puedes explorar el historial y el estado del proyecto:

# Ver el estado actual del repositorio
git status

# Ver el historial de commits
git log
git log --oneline --graph --all  # Vista compacta con gráfico de ramas

# Ver cambios de un commit específico
git show <commit-hash>

Explorar issues y releases

En GitHub, las issues son reportes de bugs o solicitudes de características. Las releases son versiones publicadas del proyecto. Puedes encontrarlas directamente en la interfaz web del repositorio, en las pestañas Issues y Releases.

💡 Para buscar issues etiquetadas como "good first issue" (ideal para contribuyentes nuevos): abre GitHub, ve a la pestaña Issues y filtra por la etiqueta correspondiente. Muchos proyectos usan esta etiqueta para facilitar la incorporación de nuevos contribuyentes. 
# Ver los tags (versiones) del repositorio local
git tag
git tag -l "v2.*"  # Filtrar tags que empiezan con v2

# Ver información de un tag
git show v1.0.0
