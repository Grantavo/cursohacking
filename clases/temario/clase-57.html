Operando con proyectos de Github [2-5]

Duración: 2:17 · Sección 2: Temario

Para contribuir a proyectos de terceros en GitHub, el flujo estándar es: hacer un fork del repositorio, clonarlo localmente, crear una rama, hacer cambios, y enviar un Pull Request (PR).

Fork en GitHub

Un fork es una copia del repositorio original en tu cuenta de GitHub. Se hace desde la interfaz web: botón Fork en la esquina superior derecha del repositorio que quieras contribuir.

# Clonar tu fork localmente
git clone https://github.com/tu-usuario/repositorio-forkeado.git
cd repositorio-forkeado

Sincronizar con el repositorio original (upstream)

Para mantener tu fork actualizado con los cambios del repositorio original, configura un remote adicional:

# Ver los remotos configurados
git remote -v

# Agregar el repositorio original como "upstream"
git remote add upstream https://github.com/usuario-original/repositorio.git

# Verificar que ahora tienes dos remotos
git remote -v
# origin    https://github.com/tu-usuario/repositorio.git (fetch/push)
# upstream  https://github.com/usuario-original/repositorio.git (fetch/push)

Actualizar tu fork

Para traer los últimos cambios del repositorio original a tu fork local:

# Bajar los cambios de upstream
git fetch upstream

# Ir a tu rama main
git checkout main

# Fusionar los cambios de upstream/main en tu main local
git merge upstream/main

# Subir los cambios actualizados a tu fork en GitHub
git push origin main

Pull Request (PR)

Una vez que tienes los cambios listos en tu fork, vas a GitHub y creas un Pull Request desde tu rama hacia la rama del repositorio original. El mantenedor del proyecto revisará tus cambios y los fusionará si son correctos.

💡 Antes de enviar un PR, asegúrate de que tu código sigue las guías de contribución del proyecto (archivo CONTRIBUTING.md) y que los tests pasan correctamente.
