Operando con proyectos de Github [3-5]

Duración: 2:40 · Sección 2: Temario

Las ramas (branches) permiten trabajar en diferentes versiones del código simultáneamente. Son la base del flujo de trabajo colaborativo en Git.

Gestión de ramas

Con git branch puedes listar, crear y eliminar ramas:

# Listar ramas locales (la actual aparece con *)
git branch

# Crear una nueva rama
git branch feature/nueva-funcion

# Cambiar a una rama
git checkout feature/nueva-funcion

# Crear y cambiar en un solo comando
git checkout -b feature/nueva-funcion

# Eliminar una rama (ya fusionada)
git branch -d feature/nueva-funcion

# Eliminar una rama (sin fusionar, forzado)
git branch -D feature/nueva-funcion

Fusionar ramas con merge

Cuando terminas de trabajar en una rama, la fusionas de vuelta a la rama principal:

# Ir a la rama destino
git checkout main

# Fusionar la rama de trabajo
git merge feature/nueva-funcion

Resolver conflictos

Si dos ramas modifican la misma parte de un archivo, Git no puede fusionar automáticamente y tendrás que resolver el conflicto manualmente:

# Git marca los conflictos en los archivos afectados
# El contenido conflictivo se muestra así:
# <<<<<<< HEAD
# versión de la rama actual
# =======
# versión de la rama que se fusiona
# >>>>>>> feature/nueva-funcion

# Después de editar manualmente el archivo, marca como resuelto
git add archivo-en-conflicto.txt
git commit -m "Resuelve conflicto en archivo-en-conflicto.txt"

Git stash

Si necesitas cambiar de rama pero no quieres hacer commit de cambios a medio hacer, usa git stash:

# Guardar cambios temporales
git stash

# Guardar cambios con un mensaje descriptivo
git stash push -m "WIP: refactor pendiente"

# Listar los stashes guardados
git stash list

# Recuperar el último stash y eliminarlo de la lista
git stash pop

# Recuperar un stash específico sin eliminarlo
git stash apply stash@{0}

# Eliminar un stash concreto
git stash drop stash@{0}
💡 El stash es muy útil cuando estás trabajando en una rama y de repente necesitas arreglar un bug urgente en main. Guardas tu progreso, arreglas el bug, y luego recuperas tu trabajo.
