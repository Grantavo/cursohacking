Comandos básicos de Linux [2-2]

Duración: 10:02 · Sección 2: Temario

Continuamos con más comandos esenciales. En esta clase verás herramientas para inspeccionar archivos, procesar texto y combinar comandos usando redirecciones y tuberías.

Visualización parcial de archivos

head y tail muestran el principio o final de un archivo:

head archivo.txt           # Primeras 10 líneas
head -n 5 archivo.txt      # Primeras 5 líneas
tail archivo.txt           # Últimas 10 líneas
tail -n 20 archivo.txt     # Últimas 20 líneas
tail -f log.txt            # Sigue el archivo en tiempo real (útil para logs)

less y more permiten navegar archivos largos de forma interactiva:

less archivo.txt           # Navegación con flechas, / para buscar, q para salir
more archivo.txt           # Versión más simple, avance con Enter/Espacio
💡 less es más potente que more: permite retroceder, buscar hacia atrás con ? y abrir múltiples archivos.

Análisis de archivos: wc, sort, uniq

wc archivo.txt             # Muestra líneas, palabras y caracteres
wc -l archivo.txt          # Solo número de líneas
sort lista.txt             # Ordena alfabéticamente
sort -n numeros.txt        # Ordena numéricamente
sort -r lista.txt          # Orden inverso
uniq repetidos.txt         # Elimina líneas adyacentes duplicadas
sort archivo.txt | uniq    # Primero ordenar, luego eliminar duplicados

Extracción y transformación: cut, tr, diff

cut -d: -f1 /etc/passwd    # Extrae el primer campo de cada línea separado por :
cut -c1-5 archivo.txt      # Extrae los caracteres 1 al 5 de cada línea
tr 'a-z' 'A-Z' < archivo   # Convierte minúsculas a mayúsculas
tr -d ' ' < archivo.txt    # Elimina espacios
diff archivo1.txt archivo2.txt  # Muestra diferencias entre dos archivos

Redirecciones: > y >>

Las redirecciones permiten enviar la salida de un comando a un archivo:

ls -la > listado.txt       # Escribe la salida en listado.txt (sobrescribe)
echo "nueva línea" >> listado.txt  # Añade al final sin sobrescribir

Tuberías: |

El operador | (pipe) conecta la salida de un comando con la entrada del siguiente:

ls -la | wc -l                             # Cuenta cuántos archivos hay
cat /var/log/syslog | grep error | head    # Busca "error" en el log y muestra primeras líneas
ps aux | grep apache                       # Busca procesos de Apache
history | grep ssh                         # Busca comandos de ssh en el historial
🔗 Combinando todo: cut -d: -f1 /etc/passwd | sort | head -n 5 — obtén los 5 primeros usuarios del sistema ordenados alfabéticamente.

Ejercicio práctico

Crea un archivo con nombres desordenados, ordénalos, cuenta cuántos hay y muestra los 3 primeros:

echo -e "root\nadmin\nroot\nuser\nnobody\nadmin\ntest" > usuarios.txt
sort usuarios.txt | uniq -c | sort -rn
cat usuarios.txt | wc -l
sort usuarios.txt | uniq | head -3
