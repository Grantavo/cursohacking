Operando con proyectos de Github [5-5]

Duración: 4:01 · Sección 2: Temario

GitHub Actions es la plataforma de CI/CD integrada en GitHub. Permite automatizar pruebas, despliegues y cualquier flujo de trabajo directamente desde el repositorio.

Concepto de CI/CD

CI (Continuous Integration) ejecuta pruebas automáticamente cada vez que haces push o un PR. CD (Continuous Deployment) despliega el código a producción si las pruebas pasan.

Estructura de un workflow

Los workflows se definen en archivos YAML dentro del directorio .github/workflows/:

# .github/workflows/tests.yml
name: Tests

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
    - name: Checkout del repositorio
      uses: actions/checkout@v4

    - name: Configurar Python
      uses: actions/setup-python@v5
      with:
        python-version: '3.12'

    - name: Instalar dependencias
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt

    - name: Ejecutar tests
      run: pytest

Workflow de deploy básico

Un pipeline que despliega a un servidor cuando se hace push a main:

# .github/workflows/deploy.yml
name: Deploy

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - name: Deploy por rsync
      env:
        SSH_KEY: ${{ secrets.SSH_PRIVATE_KEY }}
      run: |
        mkdir -p ~/.ssh
        echo "$SSH_KEY" > ~/.ssh/id_rsa
        chmod 600 ~/.ssh/id_rsa
        rsync -avz --delete ./ usuario@servidor:/var/www/app
💡 Los secretos (contraseñas, claves SSH) se configuran en GitHub: Settings → Secrets and variables → Actions. Nunca los escribas directamente en el YAML.

Triggers comunes

Los workflows se pueden activar por múltiples eventos:

on:
  push:                 # Cada vez que se hace push
  pull_request:         # Cada vez que se abre/modifica un PR
  schedule:             # Por cron (ej: cada noche)
    - cron: '0 2 * * *'
  workflow_dispatch:    # Manualmente desde la UI de GitHub
