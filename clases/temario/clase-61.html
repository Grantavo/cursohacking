Argumentos posicionales en Bash

Duración: 5:00 · Sección 2: Temario

Los argumentos posicionales permiten pasar datos a un script de Bash desde la línea de comandos. Son fundamentales para crear scripts flexibles y reutilizables.

Variables posicionales

Bash asigna automáticamente los argumentos a variables especiales:

VariableDescripciónEjemplo (./script.sh a b c)
$0Nombre del scriptscript.sh
$1Primer argumentoa
$2Segundo argumentob
$#Número de argumentos3
$@Todos los argumentos (array)a b c
$*Todos los argumentos (string)a b c
 
#!/bin/bash
# Ejemplo: mostrar-info.sh
echo "Nombre del script: $0"
echo "Primer argumento: $1"
echo "Segundo argumento: $2"
echo "Número de argumentos: $#"
echo "Todos los argumentos: $@"

# Uso: ./mostrar-info.sh hola mundo
# Salida:
# Nombre del script: ./mostrar-info.sh
# Primer argumento: hola
# Segundo argumento: mundo
# Número de argumentos: 2
# Todos los argumentos: hola mundo

El comando shift

shift desplaza los argumentos a la izquierda. El $1 se descarta, $2 pasa a ser $1, etc.:

#!/bin/bash
# Procesar argumentos en bucle con shift
while [ $# -gt 0 ]; do
  echo "Procesando: $1"
  shift
done

Validación de parámetros

Siempre valida que se hayan pasado los argumentos necesarios:

#!/bin/bash
if [ $# -lt 2 ]; then
  echo "Uso: $0 <archivo> <directorio>"
  exit 1
fi

ARCHIVO=$1
DIRECTORIO=$2

if [ ! -f "$ARCHIVO" ]; then
  echo "Error: $ARCHIVO no existe"
  exit 1
fi

Procesar opciones con getopts

Para scripts más sofisticados, getopts permite manejar banderas (-v, -o archivo, etc.):

#!/bin/bash
while getopts "vo:h" opt; do
  case $opt in
    v) VERBOSE=true ;;
    o) OUTPUT="$OPTARG" ;;
    h) echo "Uso: $0 [-v] [-o archivo]"; exit 0 ;;
    *) echo "Opción inválida"; exit 1 ;;
  esac
done
📖 En getopts, los dos puntos después de una letra (o:) indican que esa opción requiere un argumento. La variable $OPTARG contiene el valor del argumento.
