Descriptores de archivo

Duración: 6:58 · Sección 2: Temario

Los descriptores de archivo (file descriptors) son números que el sistema operativo asigna para representar archivos abiertos. Son la base de cómo Linux maneja la E/S.

¿Qué es un descriptor de archivo?

Cada proceso tiene una tabla de descriptores. Los tres primeros están predefinidos: 0 (stdin), 1 (stdout) y 2 (stderr). Cuando abres un archivo, el sistema le asigna el siguiente número disponible (3, 4, 5...).

Inspeccionar descriptores: /proc/PID/fd

ls -la /proc/$$/fd           # Muestra los descriptores del shell actual
ls -la /proc/1/fd            # Muestra los descriptores del proceso init (PID 1)
💡 $$ es una variable especial que contiene el PID del shell actual. Verás enlaces simbólicos como 0 → /dev/pts/0, 1 → /dev/pts/0, etc.

Redirigir con descriptores personalizados

Con exec puedes abrir archivos y asignarles descriptores específicos:

exec 3> salida.txt           # Abre salida.txt con descriptor 3 para escritura
echo "Hola" >&3              # Escribe usando el descriptor 3
exec 3>&-                    # Cierra el descriptor 3

Lectura con descriptores:

exec 4< archivo.txt          # Abre archivo.txt con descriptor 4 para lectura
cat <&4                      # Lee desde el descriptor 4
exec 4>&-                    # Cierra el descriptor 4

Duplicar descriptores:

exec 5>&1                    # Duplica stdout en el descriptor 5
echo "visible"               # Se ve normal
echo "invisible" >&5         # También se ve (va a stdout a través del 5)
⚠️ Los descriptores personalizados existen solo mientras dura el proceso que los creó. Al cerrar la terminal, se liberan automáticamente.

Aplicación práctica: logging

Un patrón común es redirigir toda la salida del script a un archivo de log:

#!/bin/bash
exec > log.txt 2>&1          # Todo stdout y stderr va a log.txt
echo "Iniciando proceso..."
ls /etc
echo "Proceso completado"

Límite de descriptores

Cada proceso tiene un límite de archivos abiertos. Se consulta con:

ulimit -n                    # Muestra el límite (normalmente 1024)
ulimit -n 2048               # Aumenta el límite (solo root puede subirlo arbitrariamente)
